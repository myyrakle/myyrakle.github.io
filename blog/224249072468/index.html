#LLM
[원본 링크]

superpowers는 다양한 Skill 프리셋으로 완성도 높은 개발을 도와주는 스킬 라이브러리다.
이게 주체가 되는건 아니고, 그냥 Skill의 묶음이다.

agent skill 자체가 범용적인 인터페이스인만큼, claude code, codex, gemini CLI, github Copilot, cursor 등의 에이전트들에서 전부 사용 가능하다.




기반지식

알아두면 좋은것들
https://blog.naver.com/sssang97/224062386182
https://blog.naver.com/sssang97/224096244982




컨셉

superpowers는 그냥 빠르게 찍어내는 것보다는, 체계적인 과정을 통해서 진정으로 완성된 프로그램을 만드는 것을 목표로 한다.
그래서 다음과 같은 다양한 기술들을 활용한다.

브레인스토밍
유저와의 반복적인 피드백을 통해 정확한 구현 형태를 구체화한다. 이를 위해서 UI 스켈레톤을 자동으로 만들어주기도 한다.

**worktree 기반 서브 프로젝트 생성 **
worktree와 docker 등을 활용해서 별도의 격리된 영역에서 개발 작업을 진행할 수 있다.

plan과 task 관리
브레인스토밍이 끝나고 요구사항이 완전히 분석되면, plan 과정에서 구체적은 task 단위를 정의한다.
여기서 개별 task 단위를 서브에이전트로 병렬처리할 수 있다.

TDD (optional)
빠른 셀프 테스트를 통해 안정적이고 튼튼한 소프트웨어를 만들 수 있도록 한다. 필수는 아니다.




설치

claude code를 기준으로 다룬다.
세션에서 plugin install 명령을 사용하면 즉시 설치할 수 있다.

/plugin install superpowers@claude-plugins-official

그리고 재시작하면 skillsets을 로드한 채로 실행이 되는 것이다.

그러면 기본 세팅은 끝이다. 간단하다.


그럼 이런 식으로 스킬 커맨드가 잔뜩 추가된다.
이걸 직접 호출해도 되고, LLM의 판단에 맡겨서 대신 쓰게 할 수도 있다.

다른 에이전트들도 잘 지원이 된다.
codex의 경우에는 이런 식으로 설치할 수 있다.

Fetch and follow instructions from https://raw.githubusercontent.com/obra/superpowers/refs/heads/main/.codex/INSTALL.md

이후의 사용법은 사실 거의 같다.




시작해보기: 테트리스 만들기

써보면서 이해하는게 가장 빠를 것이다.
스킬을 활용해서 웹 테트리스를 한번 만들어보자.



브레인스토밍

먼저 브레인스토밍 모드로 들어간다.
이걸 트리거하는건 어렵지 않다. 그냥 이런 요구사항을 제시하면 자동으로 브레인스토밍 스킬을 잡아서 컨텍스트를 시작해준다.

그럼 하나씩 의견을 주고받으면서 구체화할 수 있는데


이게 좋은 점 중 하나는 스켈레톤을 계속 만들면서 편리한 상호작용을 할 수 있다는 것이다.


스타일 정하고


기술스택 정하고


기능 범위도 정하고


최종 리뷰까지 완료하면, 요구사항에 대한 스펙 명세는 끝낼 수 있다.




플랜

그럼 이제 구체적인 작업 단위를 계획할 차례다.

스펙을 기반으로 플랜을 짜고

태스크 계획을 완료하면 이제 진짜 작업을 실행할 수 있다.


병렬 에이전트 모드로 실행하면 각 작업단위별 종속성에 따라 적절히 동시 작업 처리를 해준다.


작업이 완료된다면 검토해보고 마무리하면 된다.


잘 만든거같다.




디버깅

superpowers의 skill들은 단순 개발 이외에도 많은 편의성을 제공한다.

이런 단순한 디버깅이나 버그 수정 요청에 대해서도 스킬 기반의 해결책을 시도하는 것이다.
