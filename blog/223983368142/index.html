[Qdrant] group query

  • 2025.08.26
  • ::
  • 1 min read
#qdrant
[원본 링크]

qdrant도 grouping을 기반으로 하는 쿼리 구문을 제공한다.
일반 조회, scroll에서는 안되고, 전용 유사도 검색 API를 통해서만 사용할 수 있다.

복잡한 통계 기능을 제공하는건 아니고, 단순 배열 그루핑 정도만 기능한다.




데이터 세팅

적당히 깔았다.

PUT /collections/products
{
    "vectors": {
      "size": 4,
      "distance": "Dot"
    }
}

PUT /collections/products/points
{
    "points": [
      {
        "id": 1,
        "payload": {
          "color": "blue", 
          "price": 10000
        },
        "vector": [1,1,1,1]
      },
      {
        "id": 2,
        "payload": {
          "color": "red", 
          "price": 5000
        },
        "vector": [1,2,1,1]
      },
      {
        "id": 3,
        "payload": {
          "color": "blue", 
          "price": 20000
        },
        "vector": [1,1,2,2]
      },
      {
        "id": 4,
        "payload": {
          "color": "yellow", 
          "price": 4000
        },
        "vector": [1,2,3,1]
      },
      {
        "id": 5,
        "payload": {
          "color": "blue", 
          "price": 4500
        },
        "vector": [1,1,1,1]
      },
      {
        "id": 6,
        "payload": {
          "color": "black", 
          "price": 15000
        },
        "vector": [1,1,1,1]
      },
      {
        "id": 7,
        "payload": {
          "color": "white", 
          "price": 2000
        },
        "vector": [1,1,1,1]
      },
    ]
}

그러면 이런 식으로 그룹 쿼리를 날려볼 수 있다.
일반 query API와 요청 형식은 거의 같다. 그룹 기준을 지정하는 필드인 group_by가 추가될 분이다.

POST /collections/products/points/query/groups
{
  "query": [2,2,2,2], 
  "group_by": "color",
  "with_vector": true, 
  "with_payload": true
}

이러면 이제 color 기준으로 값을 묶고, hits라는 내부 필드에 모아서 뿌려준다.

참고로, 성능 문제 때문인지 그루핑해서 뿌려주는 개수에는 제한이 존재한다.
group_size라는 파라미터로 조정할 수 있고, 기본값은 3개다.
