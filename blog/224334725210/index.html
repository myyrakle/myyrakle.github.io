[LLM] z.ai와 GLM

  • 2026.07.02
  • ::
  • 4 min read
#LLM
[원본 링크]

z.ai는 중국에서 떠오른 LLM의 신흥강자 중 하나다.
회사 이름과 서비스명이 z.ai고, 모델 시리즈 이름이 GLM이다.

글을 작성하는 현 시점에서는 GLM 5.2이 최상위 모델이다.




오픈모델

z.ai는 특이하게도 모델을 완전히 오픈해놓고, 동시에 자체서비스로 팔아먹는 식으로 굴린다.
그래서 돈을 내고 API를 쓸 수도 있지만, 장비만 충분히 있다면 직접 띄워서 쓸 수도 있다.

물론 여기는 소형 모델을 지원하지는 않고, 수십-수백GB짜리의 VRAM이 필요한 대형 모델만들 제공하고 있다.
나중에는 어떨지 모르겠는데, 적당한 사이즈의 장비에서 굴리긴 어렵다.

16-64GB 사이의 적당한 모델이 필요하다면 이것보단 QWEN이나 Gemma를 사용하는 것이 좋다.




GLM 5.2

글을 쓰는 현 시점에서 최신 모델은 GLM 5.2다.

(본인들이 주장에 의하면) GPT/Claude/Gemini와 대등한 성능을 내면서 가격은 더 저렴하다.

https://z.ai/blog/glm-5.2
중국에서 Deepseek가 가성비만으로 다른 요소를 다 씹어먹는 포지션이라면, GLM은 그보단 더 비싼 대신에 훌륭한 성능을 보장하는 느낌인 것 같다.
내가 알기로 중국 자체 LLM 중에서는 지금 얘가 최고다.

Claude Opus 등에 비해서 대략 3-4배 정도 저렴한 느낌이다.




모델 종류

당연하지만 텍스트 - 텍스트 기반 모델이 주력이고

비전 모델도 있다. 이미지 기반의 해석 모델들이다.

이미지 생성 모델도 GEMINI/GPT에 비하면 밀릴테지만, 있고 비디오 생성 모델도 있다.




서비스로서의 z.ai

다른 LLM 서비스처럼 로그인 없이도 적당히 써볼 수는 있다.

현재 비로그인 상태에서 사용할 수 있는 모델은 4.7 정도다. 다른걸 쓰려면 구독을 해야한다.


Github으로도 편리하게 가입해서 사용해볼 수 있다.




비용

구독제와 종량제를 모두 지원한다.

종량제를 사용할 경우에는 다른 서비스들처럼 1M 토큰 단위로 비용을 뜯는다.

https://docs.z.ai/guides/overview/pricing
GPT/Claude의 최상위 모델이 입력 토큰을 1M당 $5~15, 출력토큰을 $15~75까지 뜯는걸 생각하면 확실히 저렴하다.

자세한 것은 요금 문서 참조: https://docs.z.ai/guides/overview/pricing

구독제를 사용할 경우에는 월/연 단위 구독제로 제공이 된다.

월간 요금

연간 요금
상대적으로 저렴하긴 한데, 막 엄청 싸다는 느낌까지는 아니다.

자세한 것은 페이지를 참조한다.
https://z.ai/subscribe

난 3달러만 결제해서 써봤다. 최소 결제 단위가 3달러더라.

후불로 뜯어가는게 아니라 선불 충전하고 다 쓰면 제한을 거는 방식이라서 비용 관리는 좋은 것 같다.


확실히 종량제로 하니까 토큰이 빠르게 녹긴 하더라




에이전트로서 활용하기

GPT/Claude 호환 API를 제공하기 때문에, Codex나 Claude 같은 에이전트에 붙여서 사용하는 것도 가능하다,

먼저 API Key를 뽑고, Z_AI_API_KEY라는 환경변수에 값을 저장한다.

[주의] 지금 codex에는 API 호환성 문제가 있다. zai API는 chat type의 응답만을 제공하는데, codex 최신버전에서는 chat 기능을 제거하고 wire_api = "responses"만을 허용하기 때문이다.
그래서 codex 구버전을 사용해야 한다.

codex를 쓰는 경우에는 다음과 같이 멀티프로필을 추가해서 사용할 수 있다.

vim ~/.codex/glm.config.toml
...

model = "glm-5.2" 
model_provider = "zai" 

[model_providers.zai]
name = "Z.ai"
base_url = "https://api.z.ai/api/paas/v4"
env_key = "Z_AI_API_KEY"
wire_api = "chat"

[profiles.glm]
model = "glm-5.2"
model_provider = "zai"

codex --profile glm


Claude Code는 이보단 간편하다.
전용 셋업 도구를 스크립트로 말아서 제공해준다.

npx @z_ai/coding-helper

그러면 이걸로 즉시 Claude <> GLM를 오갈 수 있다.
Configuration Refresh를 하면 GLM으로 세팅이 되고, Unload를 하면 다시 해제된다.




전용 에이전트: ZCode

며칠 전에 전용 에이전트 개발 도구를 공개했다.
다만 Claude Code와는 좀 다르다. CLI만 제공하는게 아니라 통합 UI를 제공하는 식으로 작동하는거라서 기존 개발 도구에 붙이는 식으로 쓸 수는 없다.
https://zcode.z.ai/en#all-downloads

리눅스는 아직 베타더라

API 키 넣어서 간단하게 연동할 수 있다.


채팅을 통해 상호작용할 수 있고


전반적인 UX 흐름은 다른 에이전트들과 크게 다르진 않다.


확실한건 더 써봐야 알겠지만, 좀 써봤을 때는 성능이 꽤 나쁘지 않은 것 같았다.
