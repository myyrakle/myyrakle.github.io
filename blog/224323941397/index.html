#PostgreSQL

PostgreSQL에서는 Rust도 확장의 형태로 사용이 가능하다. 기본은 아니고 서드파티다.
굳이 이렇게까지 쓸 일이 잘 있으려나 싶긴 한데...

현 시점에서는 아직 PG17까지만 지원되는 것 같다.

rust로 고성능 해시 처리, JSON 처리, 정규식, 빅넘버 연산 등을 최적화할 일이 있다면 쓸만은 하다.

근데 단점도 많다. 모든 crate을 사용하기는 어렵고, Rust 버전 지원도 낮다. 최신버전을 쓰기는 어렵다.
손발 다 묶인 채로 쓰는거라 Rust의 장점을 제대로 끌어내기는 요원하다.




사용 가능 여부 확인

현재 라이브러리 목록에 있는지부터 확인해야 한다. 여기 있다면 바로 쓸 수 있고, 없다면 설치를 해줘야 한다.

SHOW shared_preload_libraries;

이러면 없는거고


이러면 있는거다.




설치 (Self Hosted)

다른 확장들과 비교해서도 설치가 매우 번거로운 편이다.

권한설정같은거 해주고, Rust도 깔고 이것저것 셋업을 해줘야 한다.
Rust 정도는 깔려있다고 가정한다.

sudo chown postgres -R $(pg_config --sharedir)/extension/
sudo chown postgres -R $(pg_config --sharedir)/lib/

sudo su - postgres
source ~/.cargo/env
rustup default stable

Rust 컴포넌트와 타겟을 추가하고

rustup component add rustc-dev llvm-tools-preview
rustup target install x86_64-unknown-linux-gnu

Postgres 버전을 확인한 다음

pg_config --version

버전에 맞는 플래그를 추고 init을 돌리면 된다.

git clone https://github.com/pgcentralfoundation/plrust.git

cd ~/plrust/plrustc
./build.sh
mv ~/plrust/build/bin/plrustc ~/.cargo/bin/

cd ~/plrust/plrust
PG_VER=17 STD_TARGETS="x86_64-postgres-linux-gnu " ./build

mkdir -p ~/.pgrx
export PGRX_HOME=~/.pgrx
cargo pgrx init --pg17 $(which pg_config)

cd ~/plrust/plrust
cargo pgrx install --release --features trusted -c $(which pg_config)

그리고 오류 없이 다 통과되면 사용이 가능해질 것이다.




AWS RDS에서

다행히, 상용 클라우드들에서는 기본 확장 세트에 포함되는 경우가 흔하다.
13버전과 17버전까지는 shared_preload_libraries에 plrust를 추가하고 리로드만 하면 자동으로 설치된다.

다만, Aurora 계열에는 없는 버전이 더 많은 것 같다.

없다면 추가하고 재부팅까지 해야 한다. 다운타임이 필수적이다.

이후에 CREATE EXTENTION만 하면 된다.




확장 사용하기

설치가 잘 되어있다면, CREATE EXTENSION 명령 하나로 즉시 활성화가 가능하다.

CREATE EXTENSION plrust;


CREATE FUNCTION hello()
RETURNS text
LANGUAGE plrust
AS $$
    Ok(Some("hello".to_string()))
$$;

SELECT hello();

그리고 뭐 꼬이면 rusty한 오류가 난다.

실제로 컴파일러를 그대로 돌리는 것이기 때문이다.

그리고 파라미터나 반환값 처리도 된다.
다만 여기에는 암시적인 규칙이 몇가지 있는데

SQL 타입이 암시적으로 변환된다는 것 (예: int => i32)
모든 반환값은 Result로 감싸져야 한다는 것 (Ok 반환)
Nullable은 Option으로 표현된다는 것 정도다.


아무튼 쓰는데는 별반 문제가 없다.




오류 처리

Err에 문자열 적당히 넣어서 쏘면 표준 Postgres 형식대로 오류가 나간다.
당연히 이건 try catch같은게 없으므로 즉시 오류로 발사된다.




crate 사용

외부 crate도 제한적이지만 사용이 가능하다.
보안 문제 때문에 RDS 같은 경우에는 일부 신뢰성 있는 crate에 한해서만 사용이 허용된다.

url, regex, serde, serde_json, num-bigint 같은 것들이 그것이다. tokio 같은건 안된다.

이건 각자 문서를 확인하는 편이 좋다.
https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/PostgreSQL.Concepts.General.Using.PL_Rust.html?utm_source=chatgpt.com

그리고 이건 PL/Rust마다 문법도 많이 다르다.
일단 1.1 기준으로 적어보자면, 이렇다.

디펜던시를 함수 내에 때려박아야 한다.

CREATE OR REPLACE FUNCTION is_email(input text)
RETURNS bool
LANGUAGE plrust
AS $$
[dependencies]
regex = "1"

[code]
use regex::Regex;

let s = input.unwrap_or_default();

let re = Regex::new(
    r"^[A-Za-z0-9._%+\-]+@[A-Za-z0-9.\-]+\.[A-Za-z]{2,}$"
)?;

Ok(Some(re.is_match(&s)))
$$;

좀 못생기긴 했지만 동작은 한다.
