[Concurrency] volatile 키워드 (Java, C#)

  • 2024.09.19
  • ::
  • 3 min read
#동시성
[원본 링크]

volatile은 Java, C# 등에서 동시성 코드를 작성할 때 사용되는 주요 키워드 중 하나다.

C/C++에도 volatile 키워드가 있지만, 용도나 원리가 달라서 주요 동시성 구현 수단으로 사용되지 않는다는 것을 유의한다. 혼동하지 않아야 한다.

그리고 꽤 많은 사람들이 착각하는게 있는데, 이건 직접적으로 스레드세이프를 제공하는게 아니라 스레드세이프를 구현할 수 있는 조건 중 하나를 제공하는 것이다.

여기서는 Java를 기준으로 volatile이 동작하는 원리와 스레드 간 동기화를 구현하는 방법을 다뤄본다.




이게 정확히 무엇인가?

사실 이건 스레드와 직접적인 연관이 있는 요소는 아니다.
해당 변수의 값을 읽을때 항상 "메모리(RAM)"에서 읽어오고, 순서대로 연산하도록 강제하는 키워드다.

volatile 키워드를 사용하지 않으면, 컴파일러는 변수를 읽을때 캐시에서 읽어올 수 있는 형태로 코드를 생성한다.

https://nesoy.github.io/articles/2018-06/Java-volatile
그 이유는 당연히 빠르기 때문이다.

이게 스레드 간 동기화를 망치는 주범 중 하나다.
값을 변경하더라도 그게 캐시에 반영되는 딜레이 때문에 다른 스레드에서 잘못된 옛 값을 읽을 수 있기 때문이다.
volatile은 이러한 최적화를 제거한다.

게다가 컴파일러는 최적화 과정 중에 Reordering이라는 과정을 수행하곤 한다.
코드에서 작성한 연산 순서를 멋대로 뒤집을 수 있다는 것이다.

https://hyojun.tistory.com/entry/LeetCode-143-Reorder-List-Java
이거도 성능 최적화를 위해서 수행되는데, 싱글스레드에서는 문제없지만 멀티스레드 환경에서는 잘못된 연산 결과를 초래할 수 있다.

그래서 volatile은 이런 reordering이 발생하지 않도록 방지한다.




용도

아무튼 위에서 언급한 최적화들을 방지하기 때문에, volatile로 선언된 변수는 멀티스레드 공유와 관련해서 보장을 좀 가질 수 있다.

항상 최신의 값을 읽어오게 하기 때문에 모든 스레드가 최신 값을 실시간으로 참조하게 할 수 있다는 것이다.

그래서 volatile 변수는 하나의 스레드가 write를 하고, N개의 스레드가 read를 하는 상황에서는 안전하게 사용할 수 있다.
하지만 동시에 여러개의 thread가 write를 시도할 수 있는 상황에 대해서는 대응하지 못함을 유의한다.




예제: 가짜 Lock 만들어보기

직접 코드를 작성해서 volatile이 어떻게 동작하는지 알아보겠다.
아래는 아무런 동기화 수단도 없이 Lock을 구현한 간단한 클래스다.

그냥 lock boolean 값 하나 가져다가 기다리고 들어가고 하는 간단한 코드만 작성해놨다.


그리고 저걸 가져다가 동기화 비스무리하게 하면서 공유 변수에 접근하려 하면


매우 높은 확률로 데드락에 걸려서 프로그램이 중단될 것이다.
위에 언급했던 이유들로 인해서 스레드 캐시가 남고, 순서가 꼬이기 때문이다.

이제 volatile로 바꿔보자.

그러면 이제 수정사항이 다른 스레드에서의 접근에서도 바로 반영되기 때문에, 데드락이 발생하는 문제 자체는 해결이 된다.

다만 이걸로 제대로 된 락을 만들 수는 없기 때문에 크게 의미있는 코드는 아니긴 하다.
기대값인 50000보다 훨씬 작은 값이 나온건, 여러 스레드가 동시에 write를 하면서 동시 점유를 하기 때문이다.
위에서 언급했듯이 volatile은 1개의 스레드만이 write를 시도할때 유효하게 사용할 수 있는 기능이다.

Lock 같은 강력한 동시성을 구현할 때는 사용할 수 없지만, 한 스레드가 작업을 끝낸 뒤에 결과를 다른 스레드에 바로바로 통지할 필요가 있을 때 유용한 기능이다.


예제 코드

/*
 * This source file was generated by the Gradle 'init' task
 */
package app.src.main.java.org.example;

import java.util.ArrayList;

class FakeLock {
    boolean locked = false;

    public void lock() {
        while (locked) {
            // wait...
        }
        locked = true;
    }

    public void unlock() {
        locked = false;
    }
}

public class App {
    static int sharedCounter = 0;

    public static void main(String[] args) {
        FakeLock lock = new FakeLock();

        System.out.println("Starting threads...");

        var threads = new ArrayList<Thread>();
        for (int i = 0; i < 5; i++) {
            var thread = new Thread(() -> {
                for (int j = 0; j < 10000; j++) {
                    lock.lock();
                    sharedCounter++;
                    lock.unlock();
                }
            });
            threads.add(thread);
            thread.start();
        }

        for (var thread : threads) {
            try {
                thread.join();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }

        System.out.println("Counter value: " + sharedCounter);
    }
}
