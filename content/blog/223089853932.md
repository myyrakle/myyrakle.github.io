## 접속해보기: CloudShell

콘솔을 통해 클러스터를 만들었다면, root 계정으로 쿠버네티스 마스터 계정이 생성된다.  
그럴때는 aws 콘솔에서 shell로 들어가서 접속하는게 편하다.  
root로 로그인한 상태에서 들어가면 인증이 root로 구성되고, aws-cli나 kubectl이 다 깔려있기 때문이다.  

다음과 같이 kubeconfig를 먹여주면, kubectl로 바로 들어갈 수 있다.  

```javascript
aws eks update-kubeconfig --name 클러스터명 --region 리전명

aws eks update-kubeconfig --name test-cluster --region ap-northeast-2
```
