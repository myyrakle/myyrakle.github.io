Ingress는 HTTP 수준의 라우팅을 제어할 수 있게 해주는 고급 컴포넌트다.  
API Gateway 패턴에 대한 규격을 쿠버 자체에서 표준화해서 관리하는 것이다.  

Ingress를 실제로 연결하는 것은 Ingress Controller라는 별도 시스템의 책임이다.  
Ingress Controller의 널리 쓰이는 구현체는 nginx ingress controller(사망), istio ingress, traefik ingress controller, envoy 등이 있다.  

근데 어째서인지 쿠버네티스 그룹에서는 이걸 통째로 던지고 게이트웨이라는 새로운 기능으로 대체하려는 시도를 하고 있는 것 같다.

## 기본 계층 (Traefik)

개별 Ingress Controller들의 설치 방법에 대해 다루진 않는다.  
여기서는 Traefik Ingress Controller가 설치되어있다고 가정한다.  

만약 Traefik Ingress Controller가 잘 깔려있다면, 다음과 같은 식으로 리소스가 조회될 것이다.  

![](/images/img_fd3b76311f6ece80.png)
deployment 하나에 service 하나가 있다.  

사실 이 Ingress Controller라는 것이 그렇게 특별한 것은 아니다.  
Service와 Deployment 쌍으로 이루어진 Gateway 서버가 있고, 이게 개별 API들을 라우팅해줄 뿐이다.  

Ingress Controller의 엔드포인트는 Service를 통해 제공되는데, 다른 Service들과 동일하다.  
알아서 External IP를 뚫든 해서 개방해야 한다.

## Ingress 추가해보기 (ex: Traefik)

Ingress를 추가하려면 먼저 붙일 서비스가 있어야 한다. 여러개의 서비스를 단일 로드밸런서에 라우팅하는 것이기 때문이다.  

먼저 test-service라는 서비스를 하나 마련했다.  

![](/images/img_4ab4d60ba4341817.png)
그냥 별거 없는 단순한 서버다.  

저걸 붙이려면, 다음과 같이 Ingress를 정의해야 한다.  

```javascript
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
    name: test-ingress
    annotations:
        kubernetes.io/ingress.class: traefik
spec:
    rules:
        - http:
              paths:
                  - path: / # 라우팅 Path
                    pathType: Prefix # /* 그대로 매핑
                    backend:
                        service:
                            name: test-service # 연결할 서비스
                            port:
                                number: 80 # test-service의 포트
```
어노테이션을 통해 어떤 ingress controller를 쓸지 지정했다. 이걸 기반으로 traefik이 잡아서 라우팅을 한다.  
그리고 spec에서는 어떤 서비스를 어떤 경로로 라우팅할지를 지정한다.  

여기서는 원본 API가 루트 경로로만 동작하므로, 루트 경로로만 라우팅했다.  
만들고 나면  

![](/images/img_6704101a8bfa0e76.png)
이렇게 ingress 목록에 추가가 될 것이고  

<br>

![](/images/img_8dc36c2cfa387c20.png)
traefik의 로드밸런서로도 조회가 될 것이다.

## 특수한 동작

ingress의 spec은 표준에 정의된 부분이라서, 구현체별로 특화된 동작을 설정할 수는 없다.  
그래서 특수한 기능을 쓰려면 어노테이션을 기반으로 처리를 많이 한다.  

예를 들어, traefik controller의 경우에는 sub path로 service를 연결하더라도 path rewriting을 기본으로 해주진 않는다. 그걸 하려면 어노테이션으로 해당 기능을 활성화해야 한다.  

이런 식이다.  

```javascript
# Ingress
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
    name: test-ingress
    annotations:
        kubernetes.io/ingress.class: traefik
        traefik.ingress.kubernetes.io/router.middlewares: default-strip-test-prefix@kubernetescrd
spec:
    rules:
        - http:
              paths:
                  - path: /test
                    pathType: Prefix
                    backend:
                        service:
                            name: test-service
                            port:
                                number: 80
```

![](/images/img_0f27cd0ad2ecd8a3.png)
그러면 rewriting까지 해서 /test를 호출할때 /로 치환해줄 것이다.
