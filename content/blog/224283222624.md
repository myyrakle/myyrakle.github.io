minikube는 쿠버네티스의 초경량 배포판 중 하나다.  

이 배포판이 가진 독보적인 강점은, Linux 바깥, Windows와 MacOS에서도 구성이 가능하다는 것이다.  
k3s 같은 다른 경량 배포판들도 대부분 Linux만을 지원한다는 것을 생각하면 확실한 이점이 있는 셈이다.  

여기서는 MacOS 환경에서 minikube를 설치하는 방법을 기록한다.  

<br>

<br>

<br>

## 사전 요구사항

Docker가 미리 깔려있어야 한다.  
없다면 설치해서 실행까지 되게 해준다.  

![](/images/img_0f8d7ee7827623e2.png)

brew도 당연히 있어야 한다.  

<br>

<br>

<br>

## minikube 시작하기

brew를 통해 설치부터 한다.  

```javascript
brew install minikube
```

![](/images/img_ea0639c40388699f.png)
이게 세팅을 도와줄 도구다.  

<br>

![](/images/img_047ed72ea4a0c15a.png)
바이너리가 실행되면 설치 자체는 잘 된 것이고.  

다음과 같이 start 명령을 사용하면 쿠버네티스 서버가 초기화 및 실행된다.  

```javascript
minikube start --driver=docker
```

![](/images/img_f79739e67dab5f2e.png)

![](/images/img_1e0a8a12c467f001.png)
몇분 정도 걸린다.  

그러고 끝나면  

![](/images/img_1cbebe9fe0dc9d53.png)
kubectl이나 API를 통해 쿠버를 사용해볼 수 있을 것이다.  

이후의 사용법은 쿠버네티스 공통 사양을 따르면 된다.  
리소스 정의하고  

![](/images/img_45b4643d834df2f0.png)
넣는다.  

<br>

![](/images/img_f698f4a23770c3ec.png)

![](/images/img_83accf86be30c347.png)
그래서 컨테이너 실행까지 오류 없이 잘 되면, 세팅이 다 잘 된 것이다.  
이대로 잘 쓰면 된다.
