## 앱 구성

tokio 앱이니까 tokio는 당연히 있어야하고, console-subscriber 종속성이 추가로 필요하다.  

![](/images/img_d3c01dfa53768a42.png)
tokio의 실험적 기능인 tracing도 필요하다. 넣어준다.  

예제코드다.  

```javascript
use std::{error::Error, thread};

#[tokio::main]
async fn main() -> Result<(), Box<dyn Error>> {
    console_subscriber::init();

    loop {
        tokio::spawn(async {
            std::thread::sleep(std::time::Duration::from_secs(10));
            println!("done");
        });

        thread::sleep(std::time::Duration::from_secs(1));
    }
}
```
별건 없다.  
루프돌면서 느릿느릿하게 태스크를 하나씩 띄우고, 10초가 지나면 로그찍고 죽는게 다다.  
로직과 별개로 console_subscriber::init(); 초기 선언 한번만 해주면 tracing 서버가 뜬다. 저게 중요하다.  

그리고 다음과 같이 실행을 해준다.  

```javascript
RUSTFLAGS="--cfg tokio_unstable" cargo run
```

tokio-console을 다시 켜보면  

![](/images/img_80153c9f6a986616.png)

![](/images/img_9079ee8109235ec3.png)

여기서 모든 태스크의 현황을 실시간으로 볼 수 있다.  
실행중인 태스크는 ▶, 일시정지 상태(sleep)는 ⏸. 정지된 태스크는 ⏹ 로 표시된다.  

각각의 태스크가 뜬 줄도 뜨고, 이런저런 정보들이 꽤 표시된다.

## 다른 예제: long sleep

아래는 비정상적인 정도로 sleep을 걸고 아무것도 하지 않는 태스크를 하나 만든다.  

```javascript
use std::{error::Error, time::Duration};

use tokio::task::yield_now;

async fn long_sleeps(inc: u64) {
    let millis = inc;
    loop {
        std::thread::sleep(Duration::from_millis(millis));

        yield_now().await;
    }
}

async fn sleep_forever(inc: u64) {
    let millis = inc;
    loop {
        std::thread::sleep(Duration::from_millis(millis));
    }
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn Error>> {
    console_subscriber::init();

    let long_sleeps = tokio::task::Builder::new()
        .name("long-sleeps")
        .spawn(long_sleeps(5000))
        .unwrap();

    let sleep_forever = tokio::task::Builder::new()
        .name("sleep-forever")
        .spawn(sleep_forever(5000))
        .unwrap();

    match (long_sleeps.await, sleep_forever.await) {
        (Ok(_), Ok(_)) => println!("Success"),
        (_, _) => println!("Error awaiting tasks."),
    }

    tokio::time::sleep(Duration::from_millis(200)).await;

    Ok(())
}
```
long_sleeps도 무한으로 돌긴 하지만, yield를 통해 양보를 하는 반면 sleep_forever은 대책없이 루프를 돌며 리소스를 점유한다.

## 다른 예제: mutex deadlock

아래는 Mutex를 통해 데드락을 유발시키는 예제 코드다.  
크게 특별한 것은 없다.  

```javascript
use std::sync::{Arc, Mutex};

#[tokio::main(flavor = "multi_thread", worker_threads = 4)]
async fn main() -> Result<(), Box<dyn std::error::Error + Send + Sync>> {
    console_subscriber::init();

    let a = Arc::new(Mutex::new(0));
    let b = Arc::new(Mutex::new(0));

    {
        let a = a.clone();
        let b = b.clone();
        tokio::task::Builder::default()
            .name("A")
            .spawn(async move {
                let mut a_num = a.lock().unwrap();
                *a_num += 1;

                let delay = std::time::Duration::from_millis(1000);
                std::thread::sleep(delay);
                let mut b_num = b.lock().unwrap();
                *b_num += 1;
            })
            .unwrap();
    }

    {
        let a = a.clone();
        let b = b.clone();
        tokio::task::Builder::default()
            .name("B")
            .spawn(async move {
                let mut b_num = b.lock().unwrap();
                *b_num += 1;

                let delay = std::time::Duration::from_millis(1000);
                let mut a_num = a.lock().unwrap();
                *a_num += 1;
            })
            .unwrap();
    }

    Ok(())
}
```
서로서로 lock을 걸면서 먹통되는 전형적인 데드락 코드다.
