## FireLens 드라이버 수정

기존 서비스의 작업정의를 수정해서 덧대는 식으로 진행한다.  

![](/images/img_08bdcf448f645e58.png)

컨테이너 정의 탭으로 이동해서  

![](/images/img_94077c1bf085090e.png)

![](/images/img_83643bcc545ab05f.png)
이걸 FireLens - 사용자 지정으로 변경한다.  

<br>

![](/images/img_491ab0cf8265223e.png)
그래서 이렇게 입력하고, apikey에는 DataDog Key를 넣으면 된다.  

아래는 JSON 입력 풀버전  

```javascript
            "logConfiguration": {
                "logDriver": "awsfirelens",
                "options": {
                    "apikey": "...",
                    "provider": "ecs",
                    "dd_service": "chainshift-api",
                    "Host": "http-intake.logs.datadoghq.com",
                    "TLS": "on",
                    "dd_source": "go",
                    "dd_tags": "env:qa",
                    "Name": "datadog"
                }
            },
```

## 로그 라우팅 컨테이너 추가

그리고 추가 사이드카 컨테이너가 하나 필요하다.  
이게 실제로 로그를 받아서 외부에 전송해주는 역할을 한다.  

![](/images/img_c3a057ec6ae318b7.png)
ECR에서 공식 이미지로 제공하는게 있으니, 그걸 대충 넣기만 하면 된다.  

다음은 컨테이너 정의의 JSON 풀버전이다.  

```javascript
        {
            "name": "log_router",
            "image": "amazon/aws-for-fluent-bit:stable",
            "cpu": 0,
            "portMappings": [],
            "essential": true,
            "environment": [],
            "mountPoints": [],
            "volumesFrom": [],
            "user": "0",
            "systemControls": [],
            "firelensConfiguration": {
                "type": "fluentbit",
                "options": {
                    "enable-ecs-log-metadata": "true"
                }
            }
        }
```

그렇게 실행을 하고  

![](/images/img_5b3b9730e4642040.png)
오류 없이 잘 등록이 된다면  

<br>

![](/images/img_6a985e99b2bc3685.png)
실제로 전송까지도 잘 될 것이다.
