람다는 여러모로 정말 편리한 서비스 중 하나다.  
전반적인 관리비용이 거의 없다는 것도 그렇지만, 배포도 정말 편하긴 하다.  
그냥 아래처럼 소스만 압축해서 올리면 끝이니 말이다.  

![](/images/img_55a89be7abe56653.png)
게다가 이에 따른 새 버전 전환도 상당히 빠르고, 소스가 바뀐다고 서비스가 중간에 끊기지도 않는다.  

하지만 이렇게 그냥 무작정 올리고 마는 것은 좀.. 아쉬운 점이 있다.  
만에 하나라도 새로 올리는 소스에 문제가 있다면 어떻게 하나? 배포시에 잘 동작하는지 한번더 최종 점검을 해야 하지 않나?  

그래서 필요한 것이 [롤링 배포]라는 것이다.  
롤링 배포는 새 버전을 올리더라도 새 버전이 잘 동작할때까지 기다리고, 잘 동작하는게 확인되면 버전을 교체하되, 동작하지 않는다면 배포를 취소하는 기법이다.  

Code Deploy를 통해 이를 적용해보도록 하겠다.

## 버킷 생성

이게 다가 아니다.  
배포할때마다 버전을 어떻게 올릴지 그런걸 정의해야 하는데, 이걸 appspec 파일이라고 한다.  

그걸 저장해놓을 S3 버킷을 만들어주자.  

![](/images/img_ffc078d4121c29f2.png)

그리고 다음과 같은 형식으로 배포 설정파일을 써서 넣어주면 된다.  

![](/images/img_c8c6c3c45d447cfb.png)

```javascript
version: 0.0
Resources:
  - myLambdaFunction:
      Type: AWS::Lambda::Function
      Properties:
        Name: "express_test"
        Alias: "rolling"
        CurrentVersion: "7"
        TargetVersion: "8"
Hooks:
  - BeforeAllowTraffic: "code-deploy-check"
```
여기서 Name은 말 그대로 함수명, Alias는 아까 만든 별칭.  
Current는 현재 배포되어있는 버전. TargetVersion은 배포할 버전이다.  

그리고 BeforeAllowTraffic는 트래픽을 옮기기 전에 트리거를 걸어주는 것이다.  
값으로 들어가는건 함수명이다.  
저기서 소스의 정상작동에 대한 검증을 하면 된다.

## Health Check용 훅 함수

훅 함수는 대략 다음과 같은 구조로 작성하면 된다.  

![](/images/img_816720a0812fe277.png)

```javascript
'use strict';

const aws = require('aws-sdk');
const codedeploy = new aws.CodeDeploy({apiVersion: '2014-10-06'});

exports.handler = async (event, context, callback) => {
    //Read the DeploymentId from the event payload.
    var deploymentId = event.DeploymentId;

    //Read the LifecycleEventHookExecutionId from the event payload
    var lifecycleEventHookExecutionId = event.LifecycleEventHookExecutionId;

    /*
    ... 여기서 검증.
    */

    // Prepare the validation test results with the deploymentId and
    // the lifecycleEventHookExecutionId for CodeDeploy.
    var params = {
        deploymentId: deploymentId,
        lifecycleEventHookExecutionId: lifecycleEventHookExecutionId,
        status: 'Succeeded' // status can be 'Succeeded' or 'Failed'
    };

    //const bar = await codedeploy.getDe

    // Pass CodeDeploy the prepared validation test results.
    codedeploy.putLifecycleEventHookExecutionStatus(params, function(err, data) {
        if (err) {
            // 검증 실패
            callback('Validation test failed');
        } else {
            // 검증 성공
            callback(null, 'Validation test succeeded');
        }
    });
};
```
이건 공식문서에서 제공하는 템플릿이고, 나는 다음과 같이 사용했다.  

람다의 target version을 호출해서 오류가 나면 putLifecycleEventHookExecutionStatus로 실패했다고 이벤트를 날려주고, 문제없으면 putLifecycleEventHookExecutionStatus에 성공알림을 보내는 것이다.  

```javascript
'use strict';

const aws = require('aws-sdk');
const codedeploy = new aws.CodeDeploy({apiVersion: '2014-10-06'});
const lambda = new aws.Lambda();

exports.handler = async (event, context, callback) => {
    // 이 값은 BeforeAllowTraffic로 걸면 자동으로 들어옴
    const deploymentId = event.DeploymentId;
    const lifecycleEventHookExecutionId = event.LifecycleEventHookExecutionId;

    // 체크용 호출
    const response = await lambda
		.invoke({
			FunctionName: 'express_test:9', //버전은 배포중인 target 버전이 들어가게 해주면 됨.
			Payload: JSON.stringify({}),
		})
		.promise()

	console.log('response: ', response)

	if(response.FunctionError !=='Unhandled') {
	    console.log('성공')
	    await codedeploy.putLifecycleEventHookExecutionStatus({
            deploymentId: deploymentId,
            lifecycleEventHookExecutionId: lifecycleEventHookExecutionId,
            status: 'Succeeded'
        }).promise();

        callback(null, 'Validation test succeeded');
	} 
	else {
	    console.log('실패')
	    await codedeploy.putLifecycleEventHookExecutionStatus({
            deploymentId: deploymentId,
            lifecycleEventHookExecutionId: lifecycleEventHookExecutionId,
            status: 'Failed' 
        }).promise();

        callback('Validation test failed');
	}
};
```
람다 역할에 CodeDeploy와 LambdaInvoke 권한을 걸어주고, 메모리와 실행시간 제한을 좀 늘려줘야 한다.  

여기서 나는 단순하게 메인에서 오류가 뜨는지 아닌지만 검사했지만, 상황에 따라서 더 많은 검증을 해야할 수도 있을 것 같다.
