만약 비동기 함수를 만드는데, 비동기 함수를 호출한 부모 고루틴이 종료시점을 알아야 한다면 어떻게 하겠는가?  

여러가지 방법이 있을 수 있지만, 일반적으로 사용되는 idiom은 이러하다.  

![](/images/img_4b8352a1b7653e5b.png)
리턴값을 채널로 감싸서 생성해놓고, 그 내부에서 고루틴을 돌린 다음 그 채널값을 리턴해준다.  
그리고 고루틴이 작업을 완료하면 해당 채널에 결과값을 보내준다.  

이러한 형태로 디자인을 하면 사용자 측면에서 좀 더 그럴듯한 경험을 제공할 수 있다.  
이런식으로 말이다.  

![](/images/img_e4832cbcf658ebc1.png)
값이 필요한 시점에 <- 표현으로 블럭을 걸고 값을 받아오는 것이다.  
이러면 다른 언어들에서 흔히 제공하는 async/await과 거의 유사한 API를 제공할 수 있다.  

<br>

![](/images/img_1349c78ee599b4ef.png)
그렇다.  

전체 코드  

```javascript
package main

import (
	"fmt"
	"os"
)

func ReadFileAsync(filename string) chan string {
	result := make(chan string)

	go func() {
		if data, err := os.ReadFile(filename); err == nil {
			result <- string(data)
		} else {
			panic(err)
		}
	}()

	return result
}

func main() {
	result := <-ReadFileAsync("test.txt")
	fmt.Println(result)
}
```
