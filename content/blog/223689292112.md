## 1. AWS Lambda 배포 옵션 지원

서버를 대충 싸게 올려서 대충 쓰고 싶다면 AWS Lambda도 괜찮은 선택이 될 수 있다.  
그런 사람들을 위한 부팅 옵션을 넣어놨다.  

<br>

![](/images/img_223bb5f965e133e8.png)
Lambda 런타임은 Amazon Linux 2023, Amazone Linux 2 2가지를 모두 지원한다.  
둘 다 정상 동작하는거 확인했다.  

<br>

![](/images/img_1184c79b50e69c33.png)
feature 플래그 활성화하고  

<br>

![](/images/img_65e0f5ff954f80ee.png)
서버 시작할때 run 대신 run_on_aws_lambda를 사용하기만 하면 된다.  

<br>

![](/images/img_49918968c3e0afe5.png)
바이너리명도 bootstrap으로 맞춰주는게 좋다. 이게 미리 정해진 바이너리 이름이다.  

x86_64-unknown-linux-musl 타겟을 가져다가 명시적 컴파일을 때려야 한다.  

```javascript
rustup target add x86_64-unknown-linux-musl
cargo build --release --target x86_64-unknown-linux-musl
```

그리고 압축파일로 만들어서  

```javascript
zip -j bootstrap.zip ./target/x86_64-unknown-linux-musl/release/bootstrap
```

![](/images/img_72cbb06f7a93eb98.png)
AWS 콘솔에서 직접 올려보면 된다.  

그러면 Function URL로 바로 꽂아서 사용할 수 있다.  

![](/images/img_5a933c9887c45970.png)

![](/images/img_693db57fb6b54061.png)
