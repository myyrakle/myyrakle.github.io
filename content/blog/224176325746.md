kube-prometheus-stack은 k8s 클러스터에서 Grafana+Prometheus 기반의 리소스 모니터링을 쉽게 구성할 수 있게 해주는 숏컷 패키지다.  

어떤 노드에서 얼마나 리소스를 쓰는지, Pod들이 리소스를 얼마나 먹고있는지 등을 쉽게 파악할 수 있게 해준다.  

helm을 통해 간편하게 설치할 수 있다.  

```javascript
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm install prometheus prometheus-community/kube-prometheus-stack
```

![](/images/img_62e5ee9fb91b6d5b.png)
설치하고 나면, 다음 명령을 사용해서 정상적으로 실행이 되는지를 먼저 확인해보자.  

```javascript
kubectl --namespace default get pods -l "release=prometheus"
```

![](/images/img_35cd0b2e6780933a.png)
다 Running이면 정상이다.  

이제 Grafana 대시보드에 대한 엔드포인트를 뚫어야    한다. 온프레미스 환경인 경우에는 적당히 외부IP 열면 되고, 클라우드라면 로드밸런서를 쓰면 된다.  

```javascript
kubectl patch svc prometheus-grafana --type=json -p='[{"op": "replace", "path": "/spec/ports", "value": [{"name":"default", "port":28080, "targetPort": 3000, "protocol": "TCP"}]}]'
kubectl patch svc prometheus-grafana -p '{"spec": {"type":"ClusterIP", "externalIPs": ["192.168.0.8"]}}'
```

## 대시보드 접속

방금 열어놓은 Grafana 대시보드 Serive의 IP로 들어가면 로그인창부터 뜰 것이다.  

![](/images/img_846a839085f255d4.png)

기본 관리자 username은 admin인데, 패스워드는 랜덤이라서 시크릿을 통해 읽어와야 한다.  

```javascript
kubectl --namespace default get secrets prometheus-grafana -o jsonpath="{.data.admin-password}" | base64 -d ; echo
```

![](/images/img_4c6a878aa06c34e4.png)
이걸 패스워드로 넣고 로그인하면 된다.  

<br>
