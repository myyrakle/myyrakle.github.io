## 실시간 학습

이전 포스트에서는 CSV 파일로 대량의 학습 데이터를 초기화시키는 작업을 했었다.  
하지만 이건 초기에만 사용할 수 있는 방법이고, 지속적으로 데이터를 추가하거나 변경하는 작업은 실시간 학습을 통해 꾸준히 데이터를 넣어줘야 한다.  

사용자가 어떤 항목을 보거나 구매했는지를 기준으로 찍고 싶다면 당연히 interaction 데이터를 계속 업데이트 해줘야 하고, 상품의 정보가 바뀌거나 추가될때도 항상 업데이트를 쳐줘야 한다.  

이럴 때 사용하는 것이 이벤트 트래커를 이용한 실시간 학습이다.  

이벤트 트래거 항목에 들어가 트래커를 생성한다.  

<br>

![](/images/img_b97006670cf75dfd.png)
만드는데 있어서 딱히 뭐 선택사항이 있진 않다. 그냥 만들면 된다.  

![](/images/img_6ebdaa9b681b5c72.png)

![](/images/img_b7db89e81e1ae664.png)

그럼 저기 tracking ID라는 것이 있는데  

![](/images/img_292323259fc55802.png)
저걸 집어넣는데 사용할 것이다.  

interaction에 데이터를 추가로 집어넣는 방법은 대략 다음과 같다.  

```javascript
const aws = require('aws-sdk');

exports.handler = async (event) => {

    const personalizeEvents = new aws.PersonalizeEvents({})

    await personalizeEvents
		.putEvents({
			eventList: [
				{
				    eventType: 'VIEW', //이벤트 유형
					sentAt: new Date(), 
					itemId: '33794', //아이템 식별자
					properties: {
					    rating: "11" // RATING 필드
					}
				},
			],
			sessionId: 'test', //방문 식별용 세션값
			trackingId: 'a3c1ee52-d175-4618-b2d2-b36683d0f64b', //트래거 ID
			userId: "1", //유저 식별자
		})
		.promise()

        ......
```
비슷하게, 상품 데이터셋은 putItems. 사용자 데이터셋은 putUsers를 사용해야 한다.  
사용법은 거의 같으니 문서를 참조하길 바란다.  
[https://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/PersonalizeEvents.html](https://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/PersonalizeEvents.html)

## 상품들간의 유사도 분석: aws-sims 레시피

이번에는 상품에 대해서 한번 다뤄보겠다.  
데이터그룹에 들어가 데이터셋을 추가해준다.  

![](/images/img_01db1b315a6f1379.png)

이렇게 정의해서  

![](/images/img_0431b1061fe5a782.png)

```javascript
{
	"type": "record",
	"name": "Items",
	"namespace": "com.amazonaws.personalize.schema",
	"fields": [
		{
			"name": "ITEM_ID",
			"type": "string"
		},
		{
			"name": "TITLE",
			"type": "string"
		},
		{
			"name": "GENRE",
			"type": ["string", "null"], 
			"categorical": true
		}
	],
	"version": "1.0"
}
```

이전의 이 영화값들을 넣어줄 것이다.  

![](/images/img_d5aef5c68150e5f8.png)
여기서 장르는 일종의 카테고리라서 메타데이터로 넣어준다.  
그걸 식별해주는 값이 categorical이라는 예약된 키워드다.  
그럼 저걸로 엮인 항목들은 연관이 있는 것이라고 판단할 것이다.  

그다음에 캠페인을 또 만들고  

![](/images/img_0840014ac7c5b5f7.png)

이건 말 그대로 특정 상품과 관련된 상품들을 뽑아내주는 기능이다.

## aws-personalized-ranking 레시피

이건 특정 상품들 중에서만 추천 순위를 뽑아낼 수 있는 레시피다.  
위의 레시피들이 저장되어있는 전체 상품에서 점수를 계산했다면, 이건 딱 내가 지정한 상품들 중에서만 우열을 가릴 수 있다.

## popuarlity-count 레시피

이건 전체 사용자를 기준으로 해서 가장 인기 있는 상품을 순서대로 보여주는 레시피다.  

이런식으로 값이 뽑히는데, 스코어를 보여주진 않는다.  

![](/images/img_5233198edab2d973.png)

다른 USER_ID 값으로 뽑아도 같은 결과가 나온다.  
전체 사용자로 기준을 뽑는 것이라 그렇다.  

![](/images/img_baff82e16693c9a9.png)
근데 이럴거면 USER_ID는 왜 줘야하는건지 모르겠다.
