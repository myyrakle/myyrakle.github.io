elastic beanstalk는 웹서버를 올리기 위한 종합 패키지라고 보면 된다.  

이게 뭐 독립적인 기능은 아닌데, 기본적인 서버 구축부터 스펙 자동조절, 로드밸런싱까지 자동으로 세팅을 해주기 때문에 상당히 편리하다 할 수 있다.  
기본 소스만 올리면 나머지는 대부분 알아서 해준다.  

본 포스트에서는 간단하게 시작만 한번 해보겠다.

## 소스 업로드하기
이번엔 서버 소스를 작성해서 저기에 배포해보자.  

아래에서 사용할 언어의 샘플 코드를 받아서 서버를 작성하자.  
단순 서버만 작성하면 되는 게 아니라 빈즈톡 전용 설정파일같은 것도 좀 넣어줘야 하기 때문이다.  
[https://docs.aws.amazon.com/ko_kr/elasticbeanstalk/latest/dg/GettingStarted.DeployApp.html](https://docs.aws.amazon.com/ko_kr/elasticbeanstalk/latest/dg/GettingStarted.DeployApp.html)  

그리고 여기서 필요한 만큼 수정해서 사용하면 되겠다.  
저기서 빈스톡이 실행할 메인 소스는 app.js다 저것만 적절히 작성하면 된다.  

![](/images/img_2a76d776aa4c5511.png)

나는 대충 2개의 경로만 가지는 단순한 서버를 구현했다.  

![](/images/img_bd382990195c12cb.png)

```javascript
const port = process.env.PORT || 3000;
const fs = require("fs");

const log = function (entry) {
    fs.appendFileSync(
        "/tmp/sample-app.log",
        new Date().toISOString() + " - " + entry + "\n"
    );
};

const express = require("express");
const app = express();

app.get("/", async (req, res) => {
    log("/ 탐");
    res.json({ foo: "bar", text: "으악" });
});

app.get("/test", async (req, res) => {
    log("/test 탐");
    res.json({ foo: "test", text: "테스트" });
});

app.listen(port, () => {
    console.log("## 서버 실행");
});
```
취향껏 구성하면 된다.
