## 데이터 원천

일단 대시보드에 표시할 데이터를 어떻게 가져오는지부터 알아야한다.  
PostgreSQL 자체 데이터만으로 가져올 수 있는 것이 있고, AWS Cloudwatch에서 가져올 수 있는 것이 있다.  

예를 들어 CPU 사용량 같은 지표는 호스트에서만 관측할 수 있는 지표라서 AWS Cloudwatch의 metric API를 통해서만 가져올 수 있다. 그리고 Cloudwatch metric은 API 비용이 약간 발생한다.  

<br>

<br>

<br>

## 계정 설정

PostgreSQL에서 데이터를 가져오려면, pg_monitor 권한을 가진 계정이 필요하다.  
관측할 DB마다 별도 계정을 하나씩 다 들어가서 만들어준다.  

```javascript
CREATE USER otel_monitor WITH PASSWORD '패스워드 알아서';
GRANT CONNECT ON DATABASE chainshift TO otel_monitor;
GRANT pg_monitor TO otel_monitor;
```

![](/images/img_9632ca79a467238f.png)

그리고 Cloudwatch용 IAM 계정도 만들어서 액세스키를 받아둔다.  

![](/images/img_86d0f9accca19760.png)

```javascript
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Action": [
                "cloudwatch:GetMetricData",
                "cloudwatch:ListMetrics"
            ],
            "Effect": "Allow",
            "Resource": "*",
            "Sid": "CloudWatchMetricsRead"
        }
    ]
}
```
권한은 이것만 있으면 된다.
