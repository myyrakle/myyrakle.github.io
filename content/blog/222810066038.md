스텝평션은 대충 AWS Lambda의 보조형 서비스라고 할 수가 있다.  
다른 용도로도 쓸 수 있긴 하지만, 보통 그정도로 쓴다.  

AWS Lambda가 좋긴 좋지만 쓰다보면 여러가지 제약사항이 따를 때가 많다. 실행시간에 제한이 있어서 좀 뭔가 복잡하거나 오래걸리는 작업을 하기 힘들기 때문이다.  

스텝평션은 여러번의 Lambda 호출을 유기적으로 엮어서 관리할 수 있게 도와주는 서비스다.  
이걸 이용하면 Lambda 등을 더 효율적이고 편리하며, 다양한 용도로 응용할 수 있다.

## 상태 머신 코드

스텝펑션은 상태 머신이라는 것으로 전체적인 로직을 제어한다.  
상태머신 코드는 기본적으로 다음과 같은 구조를 가진다.  

```javascript
{
  "Comment": "...",
  "StartAt": "...",
  "States": {
     ...
  }
}
```
Comment는 말 그대로 주석, 설명이다. 중요하진 않다.  
States에는 모든 상태의 정의를 키-값 형태로 넣고, StartAt는 어떤 State에서 시작할지를 정한다.  

<br>

![](/images/img_8915d1ac532570b3.png)
만약 위와 같은 상태머신을 코드로 정의한다면  

아래와 같이 구성할 수 있다.  

```javascript
{
  "Comment": "A Hello World example demonstrating various state types of the Amazon States Language",
  "StartAt": "Pass",
  "States": {
    "Pass": {
      "Comment": "A Pass state passes its input to its output, without performing work. Pass states are useful when constructing and debugging state machines.",
      "Type": "Pass",
      "Next": "Hello World example?"
    },
    "Hello World example?": {
      "Comment": "A Choice state adds branching logic to a state machine. Choice rules can implement 16 different comparison operators, and can be combined using And, Or, and Not",
      "Type": "Choice",
      "Choices": [
        {
          "Variable": "$.IsHelloWorldExample",
          "BooleanEquals": true,
          "Next": "Yes"
        },
        {
          "Variable": "$.IsHelloWorldExample",
          "BooleanEquals": false,
          "Next": "No"
        }
      ],
      "Default": "Yes"
    },
    "Yes": {
      "Type": "Pass",
      "Next": "Wait 3 sec"
    },
    "No": {
      "Type": "Fail",
      "Cause": "Not Hello World"
    },
    "Wait 3 sec": {
      "Comment": "A Wait state delays the state machine from continuing for a specified time.",
      "Type": "Wait",
      "Seconds": 3,
      "Next": "Parallel State"
    },
    "Parallel State": {
      "Comment": "A Parallel state can be used to create parallel branches of execution in your state machine.",
      "Type": "Parallel",
      "Next": "Hello World",
      "Branches": [
        {
          "StartAt": "Hello",
          "States": {
            "Hello": {
              "Type": "Pass",
              "End": true
            }
          }
        },
        {
          "StartAt": "World",
          "States": {
            "World": {
              "Type": "Pass",
              "End": true
            }
          }
        }
      ]
    },
    "Hello World": {
      "Type": "Pass",
      "End": true
    }
  }
}
```
그리고 States에 여러가지 타입이 있는 것을 볼 수 있다.  
Pass는 다음 상태로 지나가게만 하고, Choice는 If-Else 분기를 처리한다.  
Fail은 실패를 처리하고, Wait는 대기를 처리한다.  

그리고 Pass 타입에 End 값을 주면 그대로 종료한다.  

그런데 상태머신은 GUI로 된 편리한 편집기를 제공하기 때문에, 꼭 다 알아야 하는건 아니다.  
그냥 구조가 이렇다는 것만 알아두자.

## 예제: 주사위 던지기 프로그램

나는 1-6까지의 값이 있는 주사위를 던지되, 6이 나올때까지 계속 던지는 루틴을 정의하고 싶다.  
재시도할때는 5초만 텀을 주겠다.  

그럼 이런식으로 배치를 할 수 있다.

![](/images/img_7d9e331c9dca89cd.png)

그리고 응답값으로 해당 숫자를 넣었는데, 람다에서 반환하는 값이 다음 상태에서 참조하는 값이 된다.
