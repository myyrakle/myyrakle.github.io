colima는 MacOS용 컨테이너 런타임 관리자다.  
Docker, containerd 등의 컨테이너를 MacOS에서 간편하게 구성할 수 있다.  

Linux도 되긴 하는데 굳이 쓸 이유는 없는듯.  
QEMU나 네이티브 가상화를 통해 docker를 구성해서 실행할 수 있게 해준다.  

그리고 Docker Desktop 같은 반쪽짜리와는 다르게, CLI 기반 제어가 가능하다.  
(Docker Desktop은 UI 제어만 가능함)  

<br>

<br>

<br>

## 설치

brew를 통해 쉽게 설치할 수 있다,  

```javascript
brew install colima docker docker-compose
```

![](/images/img_a1b403ac680c233d.png)

그럼 전용 CLI를 통해 런타임을 제어할 수 있을 것이다.  

![](/images/img_3782c592e6f5088d.png)

<br>

<br>

<br>

## colima 시작하기

설치되면 start로 실행하면 된다.  
가상화 스펙을 정의해야해서 넣어줄게 몇가지 있다.  

```javascript
colima start --cpu CPU개수 --memory 메모리GB --disk 디스크공간GB --vm-type-vz --mount-type=virtiofs
```

![](/images/img_b0d93feaf52a8c4e.png)
vz는 Apple 네이티브 가상화 도구인데, 이게 QEMU보다 빠르다.  

<br>

![](/images/img_4256179052ffb078.png)
그러면 가상화 인스턴스가 하나 뜨고, 그 안에서 docker가 실제로 실행되는 구조로 동작한다.  

<br>

![](/images/img_c432e40bf61be1dc.png)
그리고 docker ps 날려서 잘 뜨면 잘 세팅된 것이다.
