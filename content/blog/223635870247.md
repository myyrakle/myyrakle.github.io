zerocopy는 복사비용 없는 메모리 조작을 간편한 형태로 보조하는 crate다.  

구글이 푸크시아 제작에 쓰려고 만든걸로 알고 있다.  

<br>

<br>

## 왜 필요한가?

zero-copy 방법론에 대한 백그라운드는 별도 포스트를 참조한다.  
[https://blog.naver.com/sssang97/223629674161](https://blog.naver.com/sssang97/223629674161)  

아무튼간에 메모리에서 들고온 바이트열 데이터를 Rust에서 제대로 다루려면 역직렬화/직렬화를 거쳐서 struct/enum을 통해 가공하는게 주된 사용 패턴일 것이다.  

근데 그냥 일반적인 방법을 사용해서 직렬화와 역직렬화를 반복하면 필시 불필요한 복사가 발생할 것이다.  

이 crate는 복사가 발생하지 않는 형태의 unsafe 직렬화/역직렬화를 적절하게 감싸서 제공하는 라이브러리다.  
이것 자체로 syscall들을 제공하는 것은 아니다. 보조적인 용도다.  

<br>

<br>

<br>

## 기본 사용법

우선 zerocopy 종속성을 적절히 추가해준다.  

![](/images/img_2674d7156a62deae.png)
derive는 features로 넣는 대신에 zerocopy-derive를 따로 넣을 수도 있다.  
그러면 컴파일 속도가 나아진다나...  

아무튼, zerocopy를 사용한다면 보통 이런 느낌으로 구조체를 선언해서 사용하게 된다.  

![](/images/img_72914f3a5cf11d92.png)
실질적인 기능을 구현하는 트레잇은 총 4가지다.  
- TryFromBytes  
- FromBytes  
- IntoBytes  
- FromZeros  

TryFromBytes/FromBytes는 일반적으로 우리가 알고있는 TryFrom 패턴으로 바이트열을 구조체 등으로 변환한다. 반환 타입은 Result<&T, Err>다.  
owned가 아닌 참조형으로 구조체를 생성하기 때문에 복사 동작을 최소화한 채로 원본 데이터를 사용할 수 있다.  

IntoBytes는 간단하다. 구조체 등을 바이트열로 변환하는 것이다.  

FromZeros는 좀 특이한데, 객체가 기본값(0)인 상태가 유효한 상태임을 보장하는 trait이다.  

또 zero-copy에는 zerocopy 구현에 필요한 보장을 나타내는 마커 trait이 3가지 있다. 아래 마커들을 적절히 달아줘야 위의 기능 derive도 달성할 수 있다.  
- KnownLayout  
- Immutable  
- Unaligned  

KnownLayout는 대부분의 zerocopy 사용사례에 필요한 기본 마커다. 이걸 달아놔야 실제 구조체의 레이아웃 분석을 구현해줄 수 있다.  
Immutable는 해당 타입이 변경 불가능해야함을 뜻한다.  
Unaligned는 메모리 alignment가 필요하지 않는 타입을 가리킨다.  

<br>

일단 한번 실제로 만들어보자.  

```javascript
use zerocopy::*;

#[derive(Debug, KnownLayout, Immutable, TryFromBytes)]
#[repr(u32)]
enum Cuppa {
    Tea { iced: bool, milk: u8 } = u32::from_ne_bytes(*b"CHAI"),
    Coco { marshmallows: u8 } = u32::from_ne_bytes(*b"COCO"),
}
```
구조체 하나를 zerocopy를 덕지덕지 달면서 만들었다.  
enum에 값을 할당한건 별 의미는 없다. 그냥 한거고, 꼭 저래야 할 필요는 없다.  

이 경우엔 TryFromBytes를 달았기 때문에 바이트열->구조체 변환을 사용할 수 있다.  

그럼 이런 느낌으로 사용할 수 있다.  

![](/images/img_2eafa2feb5dec391.png)

```javascript
    let data = u64::from_ne_bytes([b'C', b'H', b'A', b'I', 1, 42, 0, 0]);

    let obj = Cuppa::try_ref_from_bytes(data.as_bytes());

    if let Ok(Cuppa::Tea { iced, milk }) = obj {
        println!("iced: {}, milk: {}", iced, milk);
    }
```
바이트열을 받아서, 그걸 변환하고 출력하도록 했다.  

<br>

![](/images/img_3116effd9ad9b3ea.png)
그럼 잘 동작할 것이다.  

앞서 말했듯이 저기서 생성된 obj 객체는 참조된 채로 구조체 모양을 갖춘 것일 뿐이다.  
그래서 실제로는 복사된 것이 아니라 참조된 것이다.  

이거 말고도 바이트열 역변환이나 자잘한 보조기능들이 있긴 하지만, 이게 zero-copy라는 crate의 핵심이다.
