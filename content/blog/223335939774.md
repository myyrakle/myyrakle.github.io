CDK는 Cloud Development Kit의 축약으로, AWS에서 직접 제공하는 IaC 도구다.  
코드로 인프라를 관리한다는 것이다. 테라폼 같은 도구들과 비교할 수 있겠다. 내부적으로는 AWS Cloudformation을 사용해서 상태 관리를 한다.  
[https://blog.naver.com/sssang97/222801645516](https://blog.naver.com/sssang97/222801645516)  

언어는 Node.js, Python, C#, Go, Java 정도가 제공되나, 파이썬을 가장 많이 쓰고 공식 레포에 예제가 많은 것도 파이썬인 것 같다. 그래서 나도 파이썬으로 하겠다.  

<br>

<br>

<br>

## cdk 기본구성

사전에 aws-cli 설치와 aws configure가 전부 되어있어야 한다.  

CLI는 npm으로 설치한다.  

```javascript
npm install -g aws-cdk
```

![](/images/img_f867f61c897ebd38.png)

![](/images/img_513e444fb9c72d20.png)
이렇게 깔고  

부트스트랩 명령을 날려서 기본 cloudformation 세팅을 해준다.  

![](/images/img_d4e3bfb622f2b93b.png)

![](/images/img_dfd523149ac75693.png)
그럼 이런식으로 중앙 구성이 생성된다.  
이건 일종의 컨트롤 플레인이다. 신경쓰지 않아도 된다.

## CDK 코드 구성

먼저 aws cdk 라이브러리 모듈을 설치한다.  

```javascript
python3 -m pip install aws-cdk-lib
```

![](/images/img_c0c407f73fa39d02.png)

그리고 나서 프로젝트를 적절히 구성한다.  

![](/images/img_8ef573ca71e77b0b.png)
cdk.json에는 실행 스크립트를 적어놓고  

소스코드에는 이런 구성을 갖춰준다.  

![](/images/img_3a28fb0e7f31447c.png)
env에 account number와 리전을 할당해주고  

<br>

![](/images/img_fcb88eb8aa76588b.png)
이런식으로 클래스를 정의해서 스택에 들어갈 리소스들을 초기화한다. 여기서는 EC2 인스턴스를 하나 생성하는 것 정도만 들어있다.  
이것 자체로는 그저 템플릿일 뿐이다. 이 스택을 생성할때 여기서 정의한 리소스들이 다 자동으로 생성이 된다.  

이렇게.  

![](/images/img_89ceb23489eeeea9.png)
마지막은 cdk 앱을 생성하고, 차례대로 스택을 생성하고, synth로 마무리를 한다. 대체로 이런 느낌으로 가면 된다.  
여기서 두번째 파라미터에 넣은 id가 스택의 고유 식별자다. 그리고 실제로 생성되는 인스턴스 이름에도 저게 다 접두사로 붙는다.  

그러고 나서 cdk deploy 명령을 사용하면 바로 리소스 배포가 시작된다.  
코드에 스택이 여러개일때는 --all 옵션을 줘서 다 실행시키거나 식별자를 직접 지정해야 하지만, 단일 스택일때는 아무 파라미터도 주지 않아도 된다.  

![](/images/img_6fdf8e8bfad69b39.png)
처음 생성할때는 뭐가 생기는지 보여주고 계속할 것인지를 물어본다.  

<br>

![](/images/img_830223394dc8cc06.png)
각 스택에 대한 cloudformation 스택이 생성되고  

<br>

![](/images/img_2fd1f048d443a3b6.png)
인스턴스도 생긴다.
