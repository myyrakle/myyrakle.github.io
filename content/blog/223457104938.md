쿠버네티스 클러스터를 제어할 일이 있다면 흔히 kubectl 도구를 사용하곤 한다.  

![](/images/img_7c900af4eaa3be25.png)

근데 사실 저건 그냥 API를 대신 쏴주는 역할일 뿐이고, 실제 클러스터의 제어는 HTTP Restful API를 통해서 이루어진다.  

![](/images/img_63dd70e86211c58e.jpg)
이렇게 말이다.  
kubectl은 인증 정보를 가져다가 api를 적당히 꾸며서 쏘고 응답을 적당히 뿌려주는 용도에 지나지 않는다.

## 프록시 열어보기

k8s api에 간편하게 접근해서 사용을 해보고 싶다면, 자체적으로 제공하는 프록시 기능을 사용하면 된다.  

```javascript
kubectl proxy
```

![](/images/img_b52839587a815877.png)
이 기능은 즉시 인증 없이 접근해서 사용할 수 있는 프록시 서버를 열어준다.  
다만, 보안상의 이유에서인지 localhost 루프백에서만 동작하게 열어주는 것 같다.  

그러면 다음과 같이 바로 찔러볼 수 있다.  

![](/images/img_590811c43ed634e4.png)
기본 경로로 찌르면 존재하는 모든 api의 경로를 뿌려준다.

## swagger 뽑아보기

k8s api 서버는 의외로 api에 대한 openapi 규격도 바로 제공을 한다.  
문서 서빙까지 해주지는 않고, 이렇게 openapi 파일 뽑아서  

```javascript
kubectl get --raw /openapi/v2  > k8s-openapi-v2.json
```

![](/images/img_52e05b253cfad025.png)

docker 같은걸 활용해서 스웨거 서버를 띄우면  

```javascript
docker run \
  -v $PWD/k8s-openapi-v2.json:/app/swagger.json \
  -p 8081:8080 \
  swaggerapi/swagger-ui
```

![](/images/img_be1887c6d417de1b.png)

바로 확인해볼 수 있다.  

![](/images/img_7dd7ff1df7cac153.png)

![](/images/img_a3f12ec62b66efca.png)
그렇다.
