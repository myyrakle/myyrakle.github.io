ArgoCD는 쿠버네티스 환경에서 주로 사용되는 코드 배포(CD) 도구다.  

이건 일반 애플리케이션 코드에 사용하는건 아니고, 쿠버네티스의 yaml 구성들을 git 기반으로 운영하고 배포할 수 있게 해주는 녀석이다.  

이런 식이다.  

![](/images/img_47800afc064791cd.png)
출처: 공식 문서  
git 기반으로 인프라 운영을 한다는 점에서 gitops라고도 부른다.  

이제 한번 구성하고 써보면서 익혀보도록 하겠다.  

<br>

<br>

<br>

## Pod 구성

먼저 쿠버에 argocd 세팅을 pod들로 배포해야 한다.  
그래야 중앙에서 통제도 하고, 제어 콘솔도 제공을 할 수 있다.  

```javascript
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

![](/images/img_bc8ba9caafdab2f9.png)

그럼 이렇게 리소스를 조회할 수가 있다.  

```javascript
kubectl get all -n argocd
```

![](/images/img_49ecd22ab4acf914.png)
뭐가 참 많다.  

다 되면 서비스의 로드밸런서 설정을 건드려서 외부로 공개가 되게 해주자.  

```javascript
kubectl patch svc argocd-server -n argocd -p '{"spec": {"type": "LoadBalancer"}}'
```

![](/images/img_d6388836896198af.png)
이래야 제어 콘솔에 접속을 할 수 있다.  

![](/images/img_62b02ea6687c3db6.png)

## 패스워드 초기화

argocd 콘솔에 로그인할 수 있는 계정과 패스워드를 만들어줘야 한다.  

우선 argocd-cli를 설치하고  

```javascript
sudo curl -sSL -o argocd-linux-amd64 https://github.com/argoproj/argo-cd/releases/latest/download/argocd-linux-amd64
sudo install -m 555 argocd-linux-amd64 /usr/local/bin/argocd
rm argocd-linux-amd64
```

![](/images/img_5446a5429390f406.png)

패스워드 초기화를 한번 때려준다.  

```javascript
argocd admin initial-password -n argocd
```

![](/images/img_b6bf3efd77a2266a.png)
그럼 최초 패스워드가 떨어지는데  

로그인하고  

![](/images/img_dbcd2fa196fc1fc3.png)

update-password로 패스워드를 변경해주면 된다.  

![](/images/img_72423833e204d79d.png)
