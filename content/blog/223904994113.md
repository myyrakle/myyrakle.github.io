pylint는 파이썬 환경의 대표적인 코드 품질 관리 도구, lint 중 하나다.  
이게 VSCode 기본 추천 도구이긴 한데, 딱히 표준은 아니고, 대안은 많다.  

린트 옵션이 다양한 편은 아니고, 포매터와의 통합이 있나...? 없는거같다.  

python 린트 환경 중에서 가장 느린 속도를 자랑한다.  

![](/images/img_ec6d6485b7a4411d.png)
이게 많이 쓰이는 이유가 딱히 완성도가 좋아서는 아닌거같다.  

설치 자체는 간단하다.  

```javascript
# pip
pip install pylint

# uv 
uv pip install pylint
```

## 기본 사용법 (CLI)

이런 코드가 있다고 하면  

```javascript
def GetFibonacciNumber(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    else:
        return GetFibonacciNumber(n-1) + GetFibonacciNumber(n-2)

# Example usage
if __name__ == "__main__":
    num = 10
    print(f"Fibonacci number at position {num} is {GetFibonacciNumber(num)}")
```

pylint를 실행할때, 린트를 돌릴 파일 경로를 넘겨주면 된다.  

```javascript
uv run -m pylint ./main.py
```

## 린트 규칙 관리 (pylintrc)

이것도 기본 규칙을 그대로 쓸 수도 있고, 규칙을 직접 정의해서 쓸 수도 있다.  
프로젝트 루트 경로에 pylintrc라는 파일을 만들고, 거기에 적당히 원하는 규칙을 세팅하면 된다.  

![](/images/img_26be1b0769efc2c0.png)

![](/images/img_524b077b8b662762.png)
그럼 실행할때 현재 경로의 rc 파일을 자동으로 인식해서 분석을 해준다.  

규칙 설정값 템플릿은 여기서  
[https://github.com/pylint-dev/pylint/blob/main/examples/pylintrc](https://github.com/pylint-dev/pylint/blob/main/examples/pylintrc)  
각 설정에 대한 의미는 아래 문서에서 확인할 수 있다.  
[https://pylint.readthedocs.io/en/stable/user_guide/configuration/all-options.html](https://pylint.readthedocs.io/en/stable/user_guide/configuration/all-options.html)
