이건 파이썬의 그... 황당한 부분 중 하나다.  

pip로 인한 모듈 설치는 전역적으로 이루어진다.  
그래서 특정 모듈의 1버전을 이미 설치한 상태에서 1.5 버전을 추가로 설치할 수가 없다. 다른 프로젝트라도.  

이런 문제를 해결하기 위해 생겨난게 가상 환경(virtual environment)라는 기능이다. 각 venv에 모듈 종속성을 별도로 설치하게 하는 것이다.  
사실 그냥 nodejs의 node_modules와 유사하다. 기본 세팅이 아닐 뿐이다.  

<br>

<br>

<br>

## 설치

venv 도구는 기본으로 딸려있을 수도 있지만, 아닐 수도 있다.  
내 경우에는 우분투 환경인데 없었다. 이런식으로 깔아주면 됐다.  

```javascript
sudo apt install python3.10-venv
```

<br>

<br>

<br>

## 가상환경 초기화

준비가 됐다면, 프로젝트에 대한 로컬 가상환경을 깔아주면 된다.  
현재 프로젝트 기준에서 상대 경로로 깔아주자. 경로는 뭐든 상관없으나, 보통 .env라는 이름으로 들어가는게 관례인 것 같다.  

```javascript
sudo python3 -m venv .env
```

![](/images/img_48071384a9ccb93a.png)

그럼 이렇게 가상환경이 초기화된다.  

![](/images/img_9e7cffb31a9471fa.png)
뭐가 많다.  

install을 행할 수 있게 현재 유저에 대해서 권한을 좀 넣어준다.  

```javascript
sudo chown -R myyrakle:myyrakle .env
```

![](/images/img_3b5c2a51e72457a7.png)

## 가상환경 활성화

가상환경을 활용하려면 가상환경을 켜서 가상환경 컨텍스트에 진입해야 한다.  

```javascript
source .env/bin/activate
```

![](/images/img_ffcc808434b0024a.png)
그럼 이렇게 터미널에 컨텍스트가 유지된다.  
이 상태에서 venv에 진입해서, 그 환경을 읽거나 쓸 수 있는 것이다.  

예를 들어 모듈을 설치하면  

![](/images/img_d9cd7602ba6ba156.png)

![](/images/img_74de665dbea6ad70.png)
venv에 설치되고, venv 환경에서만 갖다쓸 수 있게 된다.

## 가상 환경 종료

가상환경을 종료하면 이전의 컨텍스트로 돌아가고, 가상환경에서 설치된 환경들을 더이상 사용할 수 없게 된다.  

```javascript
deactivate
```
