S3는 아마존에서 제공하는 저장공간 서비스다.  
그냥 평소에 쓰던 드롭박스, 구글 클라우드 처럼 리소스 파일들을 올려두고 조회할 수 있는건데, 아마존의 서비스와 바로 연동된다는 점이 다르다.  

보통 이미지나 동영상 같은 미디어 파일들을 저장할 때 사용한다.  
좀 비싼 편이긴 하지만 감수할만한 정도다.  

기본적으로 저장한 용량에 대해서 다달이 요금이 부과되고, 파일을 올리거나 조회할때 발생하는 트래픽에 대해서도 요금이 나온다.

## Lambda로 S3에 업로드하기 (Node.js)
일단 함수를 하나 만들어보겠다.  

![](/images/img_c8e172233b35ecb3.png)

그리고 외부에서 접속할 수 있게 게이트웨이도 대충 달아준다.  

![](/images/img_cb8f5cbda109a060.png)

또, S3에 액세스할 수 있도록 권한도 부여해주자  

![](/images/img_c16b84f1ffcdbb76.png)
실행 역할에 들어가  

정책 연결  

![](/images/img_abdaeadc593675d0.png)

s3 권한을 연결해준다.  

![](/images/img_36d43a7d95f8b494.png)
그리고 귀찮은 멀티파트 파싱작업을 처리해줄 lambda-multipart-parser 모듈을 레이어로 붙여준다.  

그럼 밑작업은 대충 된 것이다.  

서버 코드는 다음과 같은 식으로 짤 수 있다.  
upload 함수에 파일명(Key)과 버킷명(Bucket), 파일데이터(Body), 접근 권한(ACL)을 지정해서 호출하면 된다.  

```javascript
const aws = require('aws-sdk');
const s3 = new aws.S3();

const parser = require('lambda-multipart-parser');

exports.handler = async (event) => {
    const {files} = await parser.parse(event);

    console.log('## 파일');
    console.log(files);

    const result = await s3.upload({
        Key: `/public/${Date.now()}.jpg`, // 파일 path
        Bucket: 'boom-bucket', // 버킷명
        Body: files?.[0]?.content, // 파일 데이터
        ContentEncoding: "base64", // 파일의 인코딩 형식. (람다는 뭐든 base64로 받아옴)
        ContentType: "image/jpeg",
        ACL: 'public-read', // 읽기는 퍼블릭
    }).promise();

    const response = {
        statusCode: 200,
        body: JSON.stringify(result),
    };

    return response;
};
```
그런데 여기서 조금 귀찮았던게, 이미지를 받을 멀티파트 데이터를 람다에선 좀 불편하게 받아온다는 것이다.  
몽땅 base64로 받는데다 파싱하기도 좀 귀찮아서 라이브러리 lambda-multipart-parser를 대충 갖다썼다.  

그리고 업로드 페이지 코드다. 별거없다.  

```javascript
<html>
    <script>
    </script>
    <body>
        <h1>테스트</h1>
        <form action="https://qdx7bkukn8.execute-api.ap-northeast-2.amazonaws.com/default/s3_test" method="POST" enctype="multipart/form-data">
            <input type="file" name="image">
            <button>파일 전송</button>
        </form>
    </body>
</html>
```
