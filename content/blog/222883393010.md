## 인증서 가져오기

AWS로 도메인을 샀다면 그냥 있는거 쓰면 되는데, 외부 도메인을 사용한다면 인증서 정보를 가져와야 한다.  

<br>

![](/images/img_b6005b98c62481fb.png)
cloudflare SSL 탭에서 [인증서 생성]을 클릭해 생성창으로 진입하고  

<br>

![](/images/img_8ce4e65463ab3bcd.png)
호스트 이름을 이제 우리가 지정하고 싶은 형태로 집어넣는다.  

<br>

![](/images/img_094aa84debc20bfc.png)
그럼 이렇게 던져준다.  

이제 저걸 ACM에 들어가서 넣어주면 된다.  

![](/images/img_3d093198e57348ca.png)
진입해서  

<br>

![](/images/img_573d40112e75f42e.png)
원본 인증서->인증서 본문  
개인 키->인증서 프라이빗 키로 넣어주면 된다.  

인증서 체인이 좀 귀찮은데.. 모든 제공자가 그런지는 모르겠는데 cloudflare는 저거 안넣으면 오류를 던지더라.  
선택사항이 아니다.  

이런 형태로 넣어주면 되는데,  

```javascript
﻿원본 인증서...

공용 인증서...
```

공용 인증서는 이걸로 다운받아서 바로 박으면 된다.  
[https://developers.cloudflare.com/ssl/static/origin_ca_rsa_root.pem](https://developers.cloudflare.com/ssl/static/origin_ca_rsa_root.pem)  

그래서 이렇게 뜨면 성공이다.  

![](/images/img_66f1874f27e3e2bc.png)
이제 저걸로 갖다붙여보자.  

<br>

<br>

<br>

## 도메인 이름 생성

도메인을 연결할 API Gateway로 이동해서, 사용자 지정 도메인 이름을 생성한다.  

![](/images/img_edbca18013a76871.png)
도메인 이름을 적어주고  

![](/images/img_7e15302118a18a8d.png)
방금 만든 인증서도 꼽아준다.  

<br>

![](/images/img_bab68bf2f1d7545c.png)
만들면 이런식으로 도메인 이름이 뜨는데, 저걸 또  

<br>

![](/images/img_a34d44bb77786fa8.png)
cloudflare에 들어가서 CNAME으로 박아준다.  

그러면 이제 거의 끝이다.  
API 매핑만 해주면 된다.  

![](/images/img_1d7e3b46c2ee533d.png)

![](/images/img_256a82a6756ffc55.png)

## 오류코드 521 (Cloudflare)

그런데 내 경우에는 할거 다 했음에도 계속 오류가 뜨는 문제가 있었다.  

![](/images/img_8dd6443a0908c9f4.png)

이 경우에는 SSL/TLS 개요에서 아래 설정이 Flexible로 되어있어서 그럴 가능성이 높다.  

![](/images/img_4a4f5f5322bea906.png)
저걸 Full(Strict)로 바꿔주면 된다.  

<br>

![](/images/img_6e8d1fd8e38d485c.png)
그럼 잘 될 것이다.
