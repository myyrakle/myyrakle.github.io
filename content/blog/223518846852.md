![](/images/img_69434f688e686b43.jpg)

Loki는 Grafana에서 만들고 관리하는 로그 수집 도구다.  
로그를 쌓고, 검색하는 기능을 제공한다. 로그 저장용 오픈소스로는 단연 최고 중 하나다.  

스토리지 옵션은 다양하다. 싱글 노드에 그냥 저장하는 방법도 있고, minio도 있고, s3 같은 클라우드 공급 스토리지를 사용해서 관리할 수도 있다.  

여기서는 단일노드에 NFS를 통해서 구성해보고, 그라파나 대시보드까지 연결해보는 것까지를 해본다.  

<br>

<br>

<br>

## 아키텍쳐

Loki는 수평 확장을 지원하는 로그 저장소다.  

![](/images/img_c4f6b5ec8050cafe.png)
[https://grafana.com/docs/loki/latest/get-started/architecture/](https://grafana.com/docs/loki/latest/get-started/architecture/)  
Loki에서 가장 중요한 개념은 인덱스와 청크다. 저 2가지의 형태로 로그 데이터를 저장하게 된다.  

청크는 말 그대로 로그를 압축해서 저장하는 형태를 말하고, 인덱스는 로그를 검색하기 위한 용도의 데이터셋을 말한다.  

인덱스는 예전에는 BoltDB라는걸 썼는데, 그건 단종했고 TSDB라는 Prometheus 원산의 포맷을 쓴다.  

자세한 내용은 다음 문서를 참조하면 좋다.  
[https://grafana.com/docs/loki/latest/get-started/architecture/](https://grafana.com/docs/loki/latest/get-started/architecture/)

## helm으로 설치하기

우선 helm으로 레포지토리를 가져온다.  

```javascript
helm repo add grafana https://grafana.github.io/helm-charts
helm repo update
```

![](/images/img_fcebb4dcb604181a.png)

그리고 구성 파일을 작성한다.  

```javascript
vi loki-single.yaml
```

![](/images/img_6c1b1a6b0fb31ee7.png)

```javascript
deploymentMode: SingleBinary
loki:
  auth_enabled: false
  ingester:
    wal:
      flush_on_shutdown: true
  commonConfig:
    replication_factor: 1
  storage:
    type: 'filesystem'
  schemaConfig:
    configs:
    - from: "2024-01-01"
      store: tsdb
      index:
        prefix: loki_index_
        period: 24h
      object_store: filesystem # we're storing on filesystem so there's no real persistence here.
      schema: v13
memberlist:
  service:
    publishNotReadyAddresses: true
singleBinary:
  replicas: 1
read:
  replicas: 0
backend:
  replicas: 0
write:
  replicas: 0
```

<br>

그리고 바로 생성을 때려준다.  

```javascript
kubectl create namespace loki
helm install --values loki-single.yaml loki grafana/loki -n loki
```

## PVC 문제 해결 (온프레미스의 경우)

클라우드 클러스터에서 쓰면 문제가 없을 수도 있는데, 온프레미스 클러스터에서 공식문서만 따라치다가는 대가리가 깨진다.  

```javascript
0/1 nodes are available: pod has unbound immediate PersistentVolumeClaims. preemption: 0/1 nodes are available: 1 Preemption is not helpful for scheduling.
```

![](/images/img_70af54aed8f739b9.png)
다음과 같은 오류가 흔하게 발생할 수 있는데  

[https://blog.naver.com/sssang97/223495801532](https://blog.naver.com/sssang97/223495801532)  
붙일 볼륨 없다고 찡찡거리는 것이다.  

내 경우에는 NFS를 쓰고 있어서 그냥 그거로 붙였다.  

![](/images/img_ec16ca34e12fe0a3.png)

```javascript
apiVersion: v1 
kind: PersistentVolume
metadata:
  labels:
    app.kubernetes.io/component: single-binary
    app.kubernetes.io/instance: loki
    app.kubernetes.io/name: loki
  name: nfs-pv-loki
  namespace: loki
spec:
  storageClassName: nfs
  capacity:
    storage: 15Gi 
  volumeMode: Filesystem
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  nfs:
    server: 192.168.1.2 
    path: /home/nfs/loki
```

![](/images/img_7db643892a440d31.png)
볼륨 만들고 레이블도 좀 달아주면 된다.  
기본 설정으로는 최소 10G짜리 볼륨이 필요하다.  

<br>

![](/images/img_ae5d2fb2f7313bc8.png)

![](/images/img_e8746a6354593ba2.png)
기본 구성에서 헬름 스택은 storage-loki라는 pvc를 만드는데, 그게 바인딩되면 잘 될 것이다.

## 직접 API 찔러서 사용해보기

한번 직접 찔러서 Loki가 잘 구성되어있는지를 검증해보자.  

![](/images/img_4528b933c993125b.png)
loki 스택은 기본적으로 ClusterIP를 통한 내부접근만을 권장하기에, 내부에서 k8s dns로 찔러서 사용하는 경우가 많다. 저대로면 pod 하나 띄우고 들어가서 써야하는데, 그러기엔 영 귀찮다.  

호스트에서 쓸 수 있도록 포트포워드좀 열고  

```javascript
kubectl port-forward --namespace loki svc/loki-gateway 3100:80
```

![](/images/img_54b09cdb7ac4e6b3.png)

API로 바로 찔러보면 된다.  

```javascript
curl "http://127.0.0.1:3100/loki/api/v1/query_range" --data-urlencode 'query={job="test"}' -H X-Scope-OrgId:foo | jq .data.result
```

![](/images/img_150fd006a928721d.png)
이건 쿼리를 검색하는 API다. 아직 아무것도 안했으니 아무것도 없다.  

<br>

```javascript
curl -H "Content-Type: application/json" -XPOST -s "http://127.0.0.1:3100/loki/api/v1/push"  \
--data-raw "{\"streams\": [{\"stream\": {\"job\": \"test\"}, \"values\": [[\"$(date +%s)000000000\", \"fizzbuzz\"]]}]}" \
-H X-Scope-OrgId:foo
```

![](/images/img_b2f689de75e35b21.png)
이렇게 json 형태로 말아서 로그를 쌓을 수 있다.  

job은 loki에서 로그를 묶는 단위다. 보통 하나의 시스템 단위를 하나의 job으로 잡고 로그를 쓸 것이다.  

![](/images/img_a3f76ee72d30e8bc.png)
그리고 다시 조회하면 밀어넣은 데이터를 쿼리해볼 수 있다.
