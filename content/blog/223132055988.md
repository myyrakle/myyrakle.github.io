consul은 쿠버네티스에서 사용할 수 있는 Service Mesh 구현체 중 하나다.  
istio에 비하면 성능도 꽤 빠르고, 사용성도 좋다는 평가가 있다.  

그리고 쿠버네티스 전용이 아니라 AWS, Azure 등 다른 서비스들과도 통합이 잘 지원된다.  

Service Discovery, K/V 스토어 기능 등을 제공한다.  

여기서는 쿠버네티스 클러스터에 consul을 구축하는 방법만 다룬다.

## consul 서버 띄우기 (GCP)

GCP GKE에 consul 서버를 띄우는 방법을 다룬다.  
GCP에서는 consul에 대한 자동화된 구성 서비스를 제공한다.  

![](/images/img_b576740a7a54428c.png)
이동해서 "구성" 누르고  

<br>

![](/images/img_9740ad2b78b9c524.png)
consul 서버를 띄울 쿠버네티스 클러스터를 선택해서, 만들어주면 된다.  
이 자동 구성은 현재는 autopilot 클러스터에는 지원이 안되고, standard 클러스터에만 지원되는 것 같다.  

그렇게 해서 다 뜬다면, 아래와 같이 리소스들이 구성이 되었을 것이다.  

![](/images/img_bcf57da40e6b80f6.png)
저기서 consule-ui라는 것이 GUI 페이지를 제공하는 서비스다.

## UI 외부 IP 열기

그냥 내부 IP만으로만 접근이 되면 상관이 없지만, 원격으로 접속해서 보고싶을 수도 있을 것이다.  
그러면 로드밸런서를 달아서 외부 공개를 해줘야 한다.  

```javascript
kubectl patch service consul-1-consul-ui -p '{"spec": {"type": "LoadBalancer"}}'
```

![](/images/img_bb902a151fd435f0.png)

그러면 IP가 열리고  

![](/images/img_0a0c8fd5d54a5aa4.png)

![](/images/img_c630fc344a86f89a.png)
접속을 할 수 있을 것이다.  

다만 이걸 그냥 공개한다는건 심각한 보안 문제가 될 수 있기 때문에, 네트워크 policy 등을 활용해서 인바운드를 제한한다거나 하는 것을 권장한다.
