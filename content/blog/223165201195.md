cobra는 대표적인 Go의 cli 제작용 도구다.  

<br>

<br>

## 설치 (CLI)

이건 생으로 다 짜려면 번거롭고 불편하게 생겨먹어서, 제공되는 cli를 써야 좀 쓸만하다.  
먼저 설치해준다.  

```javascript
go install github.com/spf13/cobra-cli@latest
```

![](/images/img_8d0e5269e233df84.png)

그래서 이렇게 실행이 되면 잘 깔린 것이다.  

![](/images/img_54b5c681015ed405.png)

<br>

<br>

<br>

## 초기화

cobra를 적용할 프로젝트 루트 경로에서 init을 먹여준다.  

![](/images/img_1c8cb633322a6fb4.png)

그럼 이렇게 뭐가 생긴다.  

![](/images/img_90e44a5daad44fe3.png)

이대로 대충 실행해보면 뭐가 되는게 없다.  

![](/images/img_f1ec2c2e8090d3ff.png)
왜냐하면  

root에 실행 로직이 없기 때문이다.  

![](/images/img_20df613108f31066.png)
저기다 Run을 추가해준다.  

그러고 실행하면  

![](/images/img_2ca5c71ff2b4ed12.png)
될 것이다.

## subcommand 추가하기

대부분의 cli 도구는 여러가지 subcommand를 제공한다.  
예를 들어, git add에서 git은 루트 커맨드고, add가 서브커맨드다.  

이것도 cli를 써서 추가해주면 된다.  
난 run이라는 서브커맨드를 추가하고 싶다.  

![](/images/img_a8a5164ee9918014.png)

그럼 이렇게 추가될 것이고  

![](/images/img_8942c54c07140c09.png)

실행도 될 것이다.  

![](/images/img_806640a43859f668.png)

그리고 -h 같은 기본 플래그 등은 자동으로 생성이 된다.  

![](/images/img_30230754b1f864bf.png)

<br>

<br>

<br>

## 플래그값 받기

git만 써봐도 git commit -m "foo" 같은 식으로 추가 옵션값을 전달하는 경우가 많다.  
이건 cobra에서 플래그라고 부른다.  

먼저 전역변수를 하나 깔아준다.  

![](/images/img_3f0f216440cb5f3e.png)
사실 나는 이것부터 좀 불편하다.  

그리고 이런식으로 플래그를 정의하고, 포인터로 값을 쑤셔박아주도록 한다.  

![](/images/img_3d1d84c23813cb9b.png)

그리고 Run에서 읽어서 쓰면 된다.  

![](/images/img_ce95dcbc092997a6.png)

그럼 추가될 것이고  

![](/images/img_8be322eafa1365ed.png)

전달도 될 것이다.  

![](/images/img_fed0213af946f758.png)
