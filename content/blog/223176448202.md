PGO는 프로그램을 실행해서 수집한 프로파일 데이터로 최적화를 돌리는 가장 심오한 최적화 기법이다.  
Go도 최근에 지원하기 시작했다. 버전 1.20부터 사용 가능하다.  

**장점**  
성능이 빨라진다. 보통은 2-4% 정도고, 극적인 경우는 아직 많지 않은 것 같다.  

**단점**  
현재 알려진 바로는, 바이너리가 오히려 더 커질 수는 있다. Rust PGO는 더 작아지던데 왜지..?  
그리고 당연하지만 빌드 시간이 오래걸린다.  

아직 나온지 얼마 되지도 않았고, 기능도 매우 부실해서 사실 굳이 쓸 이유는 없다.  
좀더 기다려야 할 것이다...

## 예제 코드

아래는 Hello API와 피보나치 계산 API가 있는 간단한 서버 코드다.  

```javascript
package main

import (
	"fmt"
	"net/http"
	_ "net/http/pprof"
	"strconv"
)

func fibonacci(n int) int {
	if n <= 1 {
		return n
	}
	return fibonacci(n-1) + fibonacci(n-2)
}

func main() {
	// 간단한 http 서버 예제
	http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintf(w, "Hello, World!")
	})

	http.HandleFunc("/fibonacci", func(w http.ResponseWriter, r *http.Request) {
		n, error := strconv.Atoi(r.URL.Query().Get("n"))

		if error != nil {
			fmt.Fprintf(w, "숫자를 입력해주세요.")
			return
		}

		result := fibonacci(n)

		fmt.Fprintf(w, fmt.Sprintf("피보나치 %d번째 수는 %d입니다.", n, result))
	})

	fmt.Println("서버 시작 중...")
	err := http.ListenAndServe(":8080", nil)
	if err != nil {
		fmt.Println("서버 시작 실패: ", err)
	}
}
```
프로파일링을 위해서 "net/http/pprof"를 import했다는게 유일한 특이점이다.
