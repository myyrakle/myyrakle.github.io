Linux는 정말 멋진 환경이지만, 단점도 꽤 많다.  

귀찮은 부분 중 하나가 배포판마다 앱 설치 환경이 다르고, 패키지 환경이 다르다는 것이다.  
그래서 설치하는 사용자 입장에서도 불편하고, 앱을 배포하는 개발자 입장에서도 배포판마다 패키지를 만드는게 좀 귀찮은 작업이다.  

AppImage는 이것을 깔끔하게 묶어서 처리해준다...고 주장하는데 사실 완벽하진 않다.  
동적 라이브러리나 기타 툴체인 같은 것들은 일부 직접 확인하고 설치해야 할 확률이 높다.  

여기서는 패키지를 만드는 것은 다루지 않고, 그냥 다운받아서 쓰는 방법만 대략 정리한다.

<br>

<br>

## 사전 종속성

AppImage는 내부적으로 fuse를 사용해서 디스크 기반 마운팅을 하는 구조를 갖고 있다.  
그래서 fuse를 설치해야 사용할 수 있다. 우분투의 경우에는 다음과 같이 설치할 수 있다.  

![](/images/img_6ead73e7c72af303.png)

```javascript
sudo apt-get install libfuse2
```

## 앱 아이콘 만들기

근데 저건 그냥 실행만 해주는거지, 실행파일을 적절한 위치에 이동시켜주거나, 앱으로 등록해주진 않는다.  
그건 각 배포판에 종속적인 방법을 써야 한다.  

일단 적절한 위치로 이동시키고  

```javascript
sudo mkdir /opt/cursor
sudo mv ~/Downloads/Cursor.AppImage /opt/cursor/cursor.AppImage
```

![](/images/img_a8bf238a63be696f.png)

우분투의 경우에는 이렇게 설정파일을 만들면 된다.  

```javascript
sudo vi /usr/share/applications/cursor.desktop
```

```javascript
[Desktop Entry]
Name=Cursor
Exec=/opt/cursor/cursor.AppImage --no-sandbox
Type=Application
Categories=Development
```

![](/images/img_c2fdf7a57e2897c9.png)

그럼 이렇게 뜨고, 실행도 잘 될 것이다.
