스레드를 사용하다보면, 이게. 항상 스레드가 풀타임으로 돌고 있지 않아도 되는 경우가 많다.  
그런데도 괜히 스레드를 돌리면 그것도 그것대로 리소스 낭비다.  

그래서 이럴때, 필요할 때만 스레드를 실행시키고 싶을때 사용하는게 바로 조건 변수(condition variable)라는 것이다.  

이건 Mutex 등과 함께 엮여서 쓰이는데, 엮여있는 Mutex의 값이 수정되고 notify가 날라올 때까지 블럭을 하는 것이 가능하다.  

아래는 스레드를 선택적으로 활성화하는 것에 대한 예제 코드다.  

```javascript
use std::time::Duration;

fn main() {
    use std::sync::{Arc, Condvar, Mutex};
    use std::thread;

    // 시작 값은 false입니다.
    let pair = Arc::new((Mutex::new(false), Condvar::new()));
    let pair2 = Arc::clone(&pair);

    // mutex의 값이 true인 동안에만 루프가 실행됩니다.
    thread::spawn(move || {
        let (lock, cvar) = &*pair2;

        loop {
            let mut available = lock.lock().unwrap();
            println!("# 스레드 실행중");

            // false라면 wait으로 대기합니다.
            // main 스레드에서 true로 변경하고 알림을 보내면 true가 되어 재활성화됩니다.
            if !*available {
                available = cvar.wait(available).unwrap();
            }
        }
    });

    // 메인 스레드
    let (lock, cvar) = &*pair;

    println!("시작");
    // 스레드 활성화
    {
        let mut available = lock.lock().unwrap();
        *available = true;
        cvar.notify_one();
    }
    std::thread::sleep(Duration::from_nanos(1));

    // 스레드 비활성화
    {
        let mut available = lock.lock().unwrap();
        *available = false;
    }
    println!("종료");
}
```

그럼 이런식으로 스레드가 잠깐 돌다가 블럭될 것이다.
