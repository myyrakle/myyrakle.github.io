람다의 함수를 url을 통해 외부에서도 호출을 가능케 하려면 Http 주소를 부여하거나 뭘 해야 한다.  
그를 위해 존재하는 것이 바로 트리거다.  

해당 함수 창에서 트리거 추가를 찾아 누르자.  
트리거는 함수를 어떻게 호출되도록 할 것인지를 정의하는 요소다.  

![](/images/img_099dce35817c33eb.png)

그리고 API 게이트웨이를 선택.  

![](/images/img_34cc3259537cbcda.png)
Http와 Rest 중에 고르면 된다. 난 Http로 골랐다.  
이 둘의 차이점에 대해서는 여기에 정리되어있다.  
https://docs.aws.amazon.com/ko_kr/apigateway/latest/developerguide/http-api-vs-rest.html  

![](/images/img_6af923dc347fe914.png)

보안 항목은 대강 '열기'로 체크해주고  

![](/images/img_59f0776968a6a3e0.png)

추가 세팅을 열어서 이름을 적어준다.  

![](/images/img_172ae41f788ee721.png)

그리고 추가하면  

![](/images/img_f9ec7f2f38b8755f.png)

함수 구성요소에 해당 게이트웨이가 추가될 것이다.  

![](/images/img_f105ba3e42afa2f6.png)
저걸 누르고 밑으로 드래그하면 이렇게 뜰텐데  

![](/images/img_ec67fee4482c5d74.png)
저 엔드포인트의 링크가 이제 접근 가능한 http url로 열린 것이다.  

저 링크로 접속하면 바로 함수가 작동된다!
