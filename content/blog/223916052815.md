ulimit은 리소스 제한을 위한 옵션 중 하나다.  
CPU와 메모리, GPU를 제외한 자질구레한 제한사항들을 모아둔 부분이다.  

예를 들어 FD 개수, 파일 최대 크기, 최대 프로세스 수 등을 조정할 수 있다.  

<br>

<br>

<br>

## Option: File Descriptor Limit

내가 봤을때 ulimit의 가장 일반적인 사용사례는 FD를 조정하는 것이다.  
가끔 DB 같은거 띄워서 빡세게 돌리다보면 FD를 과다하게 점유할 때도 있다.  

그럴때 안정적으로 FD를 사용하게 하고 싶다면, 다음과 같이 지정할 수 있다.  

```javascript
sudo docker run --ulimit nofile=1024:2048 imagename
```
이러면 SOFT limit이 1024고 HARD limit이 2048이다. 해당 컨테이너는 아무리 많이 써도 FD를 2048개까지만 열 수 있는 것이다. 그 이상을 시도한다면 오류가 발생하면서 실패한다.  

리눅스의 기본값은 SOFT와 HARD limit 모두 64000개다.
