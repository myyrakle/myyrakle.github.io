## Sparse Index

Primary Index는 기본적으로 Sparse Index다.  
행마다 인덱스가 있는게 아니라, granule(블록)마다 mark라고 하는 인덱스 단위를 구성한다.  

![](/images/img_3903087fe247b7fe.png)

그리고 MergeTree는 granule을 만들때 Primary Index 대상 컬럼의 값이 같거나 비슷한걸 뭉쳐서 저장한다. 그러니까 일치하는 행 목록을 빠르게 식별하고 스캔할 수 있는 것이다.

## 인덱스와 변경과 제한

Order Key는 접두사가 Primary Key가 되어야 한다. 그러니까, Primary Key가 (id)면 정렬키는 (id, key)로 수엉이 가능하다는 것이다. 반대로 (key, id)로 뒤집는건 안된다.  

Primary Key는 생성 후에 변경할 수 없다. 다시 처음부터 블록을 재구성해야 하기 때문이다.  

Order By는 테이블 생성 후에도 변경할 수 있으나, 제한적이다.  

```javascript
ALTER TABLE 테이블명 MODIFY ORDER BY (컬럼1, 컬럼2)
```
테이블 생성 후에 추가된 컬럼에 한해서만, append only로 붙일 수 있다.  

![](/images/img_8212902099cd8226.png)

그러니까 인덱스 구성을 바꾸고 싶다면 테이블을 생성해서 다시 붓는게 일반적이다.  
다행히도 컬럼 목록이 같나면 대충 SELECT->INSERT 쿼리로 붓는건 쉽다.  

```javascript
INSERT INTO 새테이블 SELECT * FROM 구테이블
```
