Clickhouse는 용도에 따라서 상당히 다양한 형태의 엔진을 제공한다.  

![](/images/img_f673a8c5ca13dc03.png)
이렇게 뭐가 많다. 기본 생성 옵션은 Log인 것 같다. 로그 저장하고 조회하는 용도로 만들어진 엔진이다.  

그래서인지 기능에 제한이 많다. Kafka나 Log 엔진은 컬럼의 추가나 수정같은게 지원되지 않거나 한다.  

![](/images/img_235f20b073420ee0.png)
그래서 필요에 따라서 잘 골라써야 한다.  

<br>

<br>

<br>

## 엔진 지정

엔진은 테이블 단위에 지정되는 옵션이다.  
아래와 같이 테이블 생성 DDL에 옵션을 줄 수 있다.  

```javascript
CREATE TABLE 테이블명
(
    ...
)
ENGINE = 엔진명;
```
그 외에는 일반적인 RDB들과 유사하다.  

<br>

<br>

<br>

## MergeTree 엔진

clickhouse가 자랑하는 기본급 엔진들이다. MergeTree는 일반적인 NoSQL들에서 저장 전략으로 사용하는 LSM 트리 매커니즘이라고 알면 된다.  
[https://blog.naver.com/sssang97/223232181952](https://blog.naver.com/sssang97/223232181952)  

일반적인 빅데이터 저장/검색 용도로 사용한다면 MergeTree를 선택하면 된다.  

![](/images/img_e20735fb8b3eae30.png)
몇가지 변종이 있지만 특별한 요구사항이 없다면 일반 MergeTree를 사용해도 충분하다.

## 로그 엔진

로그용 엔진들은 최소한의 기능만 포함하는 경량 엔진이다.  

![](/images/img_523e6209825a9371.png)
최초 생성 이후에 DDL을 변경하는게 불가능한 것을 포함해서 DB로서의 기능들은 좀 제한되지만, 수백만건의 쓰기를 매우 빠르게 처리할 수 있다.

## 기타 엔진

그 외에도 좀 잡다한 용도로 사용될 수 있는 특수한 형태의 엔진 타입을 제공한다.  

![](/images/img_dd6cb7375f8022ad.png)
예를 들어  
1. Memory 엔진은 진짜 메모리에 아무런 가공 없이 데이터를 올려서 쓸때 사용한다. 당연히 휘발성이다.  
2. File 엔진은 진짜 파일을 따로 만들고 그걸 clickhouse 외부에서 filesystem을 통해서 읽거나 쓸 수 있게 해준다.  

이런 것들이다. 뭐가 많다.  

각각의 엔진들에 대한 세부사항은 별도로 필요할때 정리해보겠다.
