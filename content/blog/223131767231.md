채널은 일방적인 데이터의 흐름을 표현하기에 적합한 Lock 매커니즘 중 하나다.  
내부적으로는 Mutex나 세마포어를 이용해서 구현되곤 한다. 그래서 사실 Mutex의 고수준 추상화라고 할 수도 있다. (반드시 그런 것은 아님)  

채널의 종류는 여러가지가 있지만, 일반적으로 가장 많이 사용되는 형태는 multi producer single consumer(mpsc)다.  
여러개의 프로듀서가 데이터를 보낼 수 있고, 하나의 컨슈머가 그 데이터를 다 받을 수 있는 것이다.  
이걸 그림으로 그리면 이렇다.  

![](/images/img_1c8081f7168b6673.png)
sender들은 언제든지 데이터를 보낼 수 있고, 그건 항상 채널에 큐의 형태로 저장이 된다.  
그리고 receiver는 큐에서 데이터를 하나씩 꺼내서 사용하게 되는 것이다.  

큐에 데이터가 없다면 들어올때까지 대기한다.

<br>

## 장단점

Mutex 등의 여타 Lock 구현들과 비교하면 사용법이 직관적이고 단순하다는 것이 장점이다. 일반적인 Lock을 쓸 때 흔히 겪을 수 있는 데드락 등의 문제가 현저히 적다.  

특히 Actor Model 등을 구현하기에 용이하다.  
read 양 대비 write 양이 굉장히 많을 경우에도 유리한 편이다. 당장 다 처리가 불가능하더라도 큐의 형태로 일단 다 받고 보기 때문이다.  

하지만 전반적인 성능은 Mutex만을 사용하는 것과 비교하면 떨어질 수도 있다.

## 사용례 (Rust)

Rust에서 간단한 채널 기반의 ping-pong 프로그램을 구현한 예다.  

먼저 컨슈머가 전달받을 이벤트의 형태를 정의한다.  

![](/images/img_6c74b4b5f0d0894e.png)

그리고 그걸 기반으로 채널 객체를 생성한다.  

![](/images/img_8bf1255c4f8352af.png)
여기에는 sender와 receiver가 자동으로 생성된다.  
sender를 producuer가 나눠서 가지면 되고, receiver는 단일 consumer만 가질 수 있다.  

그럼 이제 저걸 가져다가 동기화를 할 수 있다.  
다음은 sender가 전달하는 명령을 기반으로 문자열을 출력하고 리시버를 종료하는 로직을 구현한다.  

![](/images/img_d526d93b2c00d7b2.png)

그럼 기대한대로 실행될 것이다.  

![](/images/img_338a63cad18dcfc0.png)

전체 코드다.  

```javascript
use tokio::sync::mpsc;

enum Event {
    Message(String),
    Terminate,
}

#[tokio::main]
async fn main() {
    let (sender, mut receiver) = mpsc::channel(100);

    // receiver
    tokio::spawn(async move {
        loop {
            let event = receiver.recv().await.unwrap();
            match event {
                Event::Message(msg) => {
                    println!("message: {}", msg);
                }
                Event::Terminate => {
                    println!("terminate");
                    break;
                }
            }
        }
    });

    // sender
    tokio::spawn(async move {
        sender.send(Event::Message("hello".to_string())).await;
        sender.send(Event::Message("world".to_string())).await;
        sender.send(Event::Terminate).await;
    });

    loop {}
}
```
