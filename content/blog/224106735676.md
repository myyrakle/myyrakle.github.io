시스템 소프트웨어를 업데이트하고 나서부터 갑자기 일종의 크래시가 발생하기 시작했다.  
컴퓨터를 오래 켜뒀다 돌아올때마다 그냥 검은 화면만 뜬 채로 뻗어버린 것이다. 강제로 재부팅을 해야만 다시 사용할 수 있었다.  

![](/images/img_7c60e464e5993e1e.png)

![](/images/img_9e06e128ddbb0a5d.png)

포럼을 찾아보니까, Arch Linux 계열, AMD GPU를 쓰는 경우에 공통적으로 발생하는 문제인 것 같다.  
다른 사람들은 부팅조차 안되는 경우가 있는 것 같은데, 내 경우에는 발생 빈도가 매우 낮았다. 부팅 자체는 문제가 없다. 갑자기 뻗을 뿐이지.  

내가 일단 사용한 방법은 커널 매개변수를 조정하는 것이었다.  

```javascript
sudo vim /boot/loader/entries/linux-cachyos.conf
```

![](/images/img_bd65e7b31e939f38.png)

```javascript
amdgpu.ppfeaturemask=0xffffffff # 전원 관리 기능 활성화 
amdgpu.gpu_recovery=1 # 복구 활성화
```
gpu_recovery 이 옵션을 켜면 충돌이 났을때 자동으로 복구를 시도한다.  
이거 켜니까 가끔 순간적으로 뻗긴 하는데, 다시 몇초 안에 정상화를 해준다.  

![](/images/img_b5a7b18f1d0c37b2.png)

이거 말고는 일부 기능을 강제로 비활성화하는 방법도 있다.  

```javascript
amdgpu.dc=0
```
이러면 Display Core라는 걸 비활성화해서 예전 구버전 모드로 동작한다고 하는데, 해보니까 문제가 사라지긴 하더라.
