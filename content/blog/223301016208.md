## 모듈 가져오기

가져올때는 마찬가지로 go get 같은걸로 가져오면 되는데...  

![](/images/img_b42e11e617f559d3.png)

![](/images/img_6d76cb3fb864297b.png)
보통은 그냥 되지 않는다.  

이건 golang의 프록시 구조에 기인한다.  

![](/images/img_50a7659bc2dd4491.png)
go-cli는 github에서 코드를 가져온다고 해서 직접 github에 접속하지는 않는다.  
go-proxy라고 하는 탈중앙화스러운 중앙화 레지스트리에 접근해서, 그걸 통해서 코드를 가져오기 때문이다.  

문제는, 이게 본질적으로 public 모듈들에만 사용할 수 있는 것이란 점이다.  
저 프록시란 놈은 go-cli를 통해 로드된 코드들을 공개 인터넷에 박제하는 것을 업으로 삼는다.  
모두에게 공개된다 이말이다.  

그래서 private하게 내부적으로만 쓰려면 중간에서 프록시를 날려버려야 한다.  

일단 그러려면 먼저 ssh로 접근이 가능하게끔 설정을 해줘야 하는데. ~/.gitconfig를 찾아서 이런식으로 추가를 하거나  

```javascript
[url "git@github.com:myyrakle/go-privatge.git"]
 insteadOf = https://github.com/myyrakle/go-private
```

git-cli로 바로 수정을 해줘도 된다.  

```javascript
git config --global url."git@github.com:myyrakle/go-privatge.git".insteadOf "https://github.com/myyrakle/go-private"
```

그리고 모듈을 가져올 때는 GOPRIVATE이란 환경변수에 github 조직 경로를 넣어주고 실행하면 된다.  

```javascript
$env:GOPRIVATE = "github.com/myyrakle"; go get github.com/myyrakle/go-private@v0.1.0
```
이건 windows  

<br>

```javascript
GOPRIVATE=github.com/myyrakle go get github.com/myyrakle/go-private@v0.1.0
```
이건 linux  

이러면 go-proxy를 끄고, 저장되어있는 github credential을 통해 github에 ssh로 접근해서 코드를 따온다.  

![](/images/img_d6520824ff522e10.png)
그럼 이제 잘 받아지는 것을 볼 수 있을 것이다.

## 폐쇄망?

golang은 폐쇄망에서 쓰기가 좀 끔찍한 편이다. 물론 원래 폐쇄망이란거 자체가 끔찍한 현상이긴 하다.  

다른 언어들은 모듈 레지스트리를 사설로 띄우기도 하는데, golang은 명확하게 중앙화된 레지스트리도 없고, 그래서 그걸 포크떠서 로컬에 띄우기도 애매하다. 일단 그런 접근법은 좀 고통스러울 것이다.  

그나마 나은 방법은 vendor 기능을 사용해서 모듈을 전부 로컬에 다운받고 통째로 옮겨서 쓰는 것이다.  

<br>

![](/images/img_dbfc1260b42ac9aa.png)
go mod vendor를 쓰면  

<br>

![](/images/img_b65787275708cf5d.png)
이렇게 로컬에 파일을 다운받는다.  

<br>

![](/images/img_0e631eafa2d7e2f4.png)
펼쳐보면 쓰던게 다 들어있다.  

<br>

![](/images/img_5bb0ecac4600c3f6.png)
그리고 modules.txt라는 통짜 텍스트파일을 매니페스트로 생성하는데, go mod 시스템은 이걸 읽어서 버전이 일치한다면 종속성으로 사용한다.  

이 상태로 .../go/pkg 경로의 다운받은 모듈을 전부 날리고 실행해도  

![](/images/img_ac7807c0a80f0a47.png)
vendor의 디펜던시를 읽어서 잘 실행될 것이다.
