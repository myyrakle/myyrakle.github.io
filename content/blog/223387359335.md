## 기본구성

전역변수나 전역에 가까운 라이프타임을 가진 변수로 서킷브레이거 변수를 둔다.  

![](/images/img_cacb60efb689adbe.png)
그리고 Setting 객체를 통해 설정값들을 넣고 서킷브레이커 객체를 생성하면 된다.  

세팅에는 이런 값들이 있다.  

![](/images/img_8a5d1920ec0e03b5.png)
1. Name은 메타데이터성 이름이고.  
2. MaxRequets는 half-open 상태일때 한번에 들어올 수 있는 요청 수다.  
3. Interval은 close 상태에서 내부 카운트를 초기화하기 위한 주기다. 0이면 초기화하지 않는다.  
4. Timeout은 말 그대로 타임아웃 설정이다. 해당 초가 지나면 half-open 상태가 된다. 0이면 60초다.  
5. ReadyToTrip은 close 상태로 진입하기 위한 조건이다. 저 반환값이 true가 되면 close가 된다.  
6. OnStateChange는 그냥 상태가 변경될때마다 호출된다.  
7. IsSuccessful는 성공 조건을 지정한다. error가 반환될때 특정 오류만 성공으로 간주해야 하거든 사용하면 된다.  

아래는 서킷브레이커 기반으로 Get 요청을 날리는 함수를 구현한 예다.  

![](/images/img_f08796591f51db27.png)
별건 없다. Execute로 감싸서 실행하기만 하면 된다.  
저 내부 함수에서 오류를 반환하면 그게 ReadyToTrip으로 검사해서 close 상태로 진입하고, 그런 식으로 제어가 되는 것이다.  

<br>

![](/images/img_4088f8dd9433c9a6.png)
그러면 그냥 쓰면 된다.  

<br>

아래는 전체 예제 코드다.  

```javascript
package main

import (
	"fmt"
	"io"
	"log"
	"net/http"

	"github.com/sony/gobreaker"
)

var circuitBreaker *gobreaker.CircuitBreaker

func init() {
	var setting gobreaker.Settings
	setting.Name = "HTTP GET 테스트"
	setting.ReadyToTrip = func(counts gobreaker.Counts) bool {
		// 3번 이상 호출되고, 60% 이상 실패하면 circuit breaker가 열린다.
		failureRatio := float64(counts.TotalFailures) / float64(counts.Requests)
		return counts.Requests >= 3 && failureRatio >= 0.6
	}

	circuitBreaker = gobreaker.NewCircuitBreaker(setting)
}

// Get wraps http.Get in CircuitBreaker.
func Get(url string) ([]byte, error) {
	// 서킷브레이커를 통해 호출한다.
	body, err := circuitBreaker.Execute(func() (interface{}, error) {
		resp, err := http.Get(url)
		if err != nil {
			return nil, err
		}

		defer resp.Body.Close()
		body, err := io.ReadAll(resp.Body)
		if err != nil {
			return nil, err
		}

		return body, nil
	})
	if err != nil {
		return nil, err
	}

	return body.([]byte), nil
}

func main() {
	body, err := Get("http://www.google.com/robots.txt")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println(string(body))
}
```
