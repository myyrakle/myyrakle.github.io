쿠버 클러스터를 설정하다보면 이런 오류를 목도할 수가 있다.  

```javascript
Error: failed to run Kubelet: running with swap on is not supported
```

![](/images/img_19f4278d6028a51d.png)
스왑을 지원하지 않는다는 것이다.  
스왑은 우리가 아는 스왑이 맞다. 디스크를 메모리처럼 쓰는 스왑 메모리 말이다.  

쿠버는 기본적으로 스왑에 대한 기능을 지원하지 않기 때문에, 따로 꺼줘야 한다. 그게 권장사항이다.  
스왑을 쓰면 성능에 대한 보장도 안되고, 중요도가 낮기 때문에 쿠버 팀에서는 굳이 구현하려 들지 않았다고 한다.  

1.28부터는 베타테스트로 스왑을 지원하나, 정식 기능이라 하기는 어렵다.  

아무튼 그냥 스왑을 끄려면 이렇게 끄면 된다.  

```javascript
sudo swapoff -a
```

![](/images/img_fc151ae1761274f4.png)
그러면 작동할 것이다.  

근데 저건 컴퓨터를 재부팅하면 다시 활성화될 수 있다.  
가장 좋은 지속적인 솔루션은 아래 파일을 수정해서  

```javascript
sudo vi /etc/fstab
```

![](/images/img_600873c3a99be1c7.png)

![](/images/img_8cd0504960fe3be6.png)
저걸 가리는 것이다.
