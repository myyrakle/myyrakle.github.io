시스템을 만들고 배포하다보면, 은근히 불편한 것 중 하나가, 패스워드, 비밀키 같은 비밀스런 설정값을 집어넣는 것이다.  
보통 이런건 .env 같은 환경변수 파일을 두고 사용할 경우가 많은데, Parameter Store는 이걸 조금 대체하기 위한 기능이라고 할 수 있다.  

이건 키-값형태로 하나씩 전역적인 값을 두고 런타임에 호출해 꺼내오는 방식을 취한다.  
env와 이 방식 모두 각자 장단이 있는 것 같다.

## 파라미터 만들기

만드는건 쉽다.  
그냥 이름과 값을 잘 지어주면 된다.  

![](/images/img_2d5e567f0582ad87.png)

그럼 이렇게 만들어질 것이다.  

![](/images/img_b01b0b3aedd6ab52.png)
다만 한 이게 리전에 몰빵이 되어있는 형태라, 리전 안에 있는 시스템들끼리 이름이 미묘하게 겹칠 수도 있다.  
그럴 경우에는 이름에 접두사를 요령껏 잘 붙여서 관리를 해줘야 한다.  

보통 URL 패턴을 사용해서 /시스템명/PASSWORD 같은 식으로 많이들 쓴다.

## 파라미터 사용하기

저 저장한 값을 가져오려면 aws sdk를 사용하면 된다.  

아래는 AWS Lambda Node.js에서 파라미터값을 가져오는 예시 코드다.  

```javascript
const aws = require('aws-sdk');
const ssm = new aws.SSM();

exports.handler = async (event) => {
    const DATABASE_URL = await ssm.getParameter({
        Name: 'DATABASE_URL',
    }).promise();

    console.log(`DATABASE_URL: ${DATABASE_URL.Parameter}`)

    // TODO implement
    const response = {
        statusCode: 200,
        body: JSON.stringify('Hello from Lambda!'),
    };
    return response;
};
```
