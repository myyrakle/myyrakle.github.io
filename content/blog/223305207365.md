#### 1. Provider 정의 숏컷 추가

기존에는 DI에 프로바이더를 주입하려면, DI 사이클 처리를 위한 IProvider 구현체를 직접 정의해주고  

![](/images/img_c9406447789207ca.png)
디펜던시까지 직접 정의한 후에, di_context에서 직접 의존성을 꺼내와야 했다.  

저걸 하나의 함수 정의로 축약하기 위한 숏컷 구문을 이제는 제공한다.  

![](/images/img_599bb35e762c3884.png)
이런식이다.  
파라미터에 필요한 종속성을 열거하면, di_context에서 자동으로 가져온다. (그렇게 보이게끔 변환한다.)  

<br>

![](/images/img_6a9c12efaa4e75a0.png)
등록할땐 이렇게 하면 된다.  
좀더 세련되게 꾸밀 필요는 있을 것 같다.  

그리고 좀더 스프링스러운 alias들도 제공한다.

#### 2. Request ID 자동주입기능 추가

추후의 로그 추적 등을 위해서 request-id가 요청마다 자동 생성되도록 구성했다.  

![](/images/img_4dc69735fab01ce1.png)

![](/images/img_884783f92fcbb672.png)
클라이언트가 임의로 request-id를 보낸다면 자동 생성하지 않고 보내준걸 그대로 전달한다.
