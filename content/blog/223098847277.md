![](/images/img_5b6b5229a346b053.png)
Fargate는 AWS에서 제공하는 멋진 기반서비스 중 하나다.  
Docker 기반 컨테이너를 다루는 시스템이라면, 대부분 Fargate를 통해 구동된다.  

하지만 Fargate의 가장 큰 단점은.. 비용이다.  
EC2를 띄워서 사용하는 것에 비하면 조금 비싼 편이기 때문이다.  

그래서 AWS는 2020년부터 Fargate에 Spot 요금제를 추가했다.  
이게 뭐냐면, 더 싼 가격에 제공하는 대신에 AWS에서 자원이 모자랄 경우에 "강제로 종료될 수 있다."는 것이다.  

그래도 리스크를 감수할만큼 가격이 꽤 싸긴 하다.  

![](/images/img_cb207bdcc9ae3eff.png)
기본비용이 이런데  

<br>

![](/images/img_043cb3f814a64bcc.png)
Spot을 쓰면 3배 정도 싸다.  

만약 vcpu 한개와 4기가 메모리를 쓴다면,  
기본 Farate는 한달에 (0.04656 * 24 * 30) + (0.001533 * 4 * 24 * 30) => 37.93824달러 = 50000원이고  
Spot은 (0.013968 * 24 * 30) + (0.001533 * 4 * 24 * 30) => 14.472달러 = 19000원이 드는 셈이다.  

거의 3배 가까이 싸다.  

<br>

<br>

<br>

## 사용법

먼저 ECS 클러스터 단위에서 공급자 전략을 수정해줘야 한다.  

![](/images/img_d4e09dbb10e416af.png)
이런식으로 추가할 수 있는데, 아무래도 통째로 Spot을 띄우는건 좀 위험한 구성이다보니  

<br>

![](/images/img_126c2e358e11ced4.png)
이런식으로 섞어서 쓰는게 좋다. 가중치가 높을수록 퍼센트가 높아지는 것이다.  
이러면 인스턴스가 2개 필요할때, 하나는 Spot, 하나는 기본으로 띄워서 혹시라도 SPOT이 죽어도 전부 다 죽지는 않게 된다.  

그리고 SPOT이 강제로 죽더라도, ECS가 그 빈자리를 기본 Fargate로 다시 프로비저닝해줄 것이다.  

서비스를 생성할 때도 클러스터의 기본값을 따르도록 한다.  

![](/images/img_b1763226238737b7.png)

이미지는 그냥 nginx 갖다썼다.  

![](/images/img_26d568e7041dec95.png)
그리고 태스크는 2개만 띄웠다.  

<br>

![](/images/img_b8e924755092e405.png)
가중치를 1:1로 설정했으니, 특별한 상황이 아니라면  

<br>

![](/images/img_ea85080a6f5ef92e.png)
하나는 SPOT  

<br>

![](/images/img_8e5cdbbb217ff3ba.png)

## 알림받기

근데 뭐 ECS 스케일링으로 어떻게 터지는 상황을 면피할 수는 있다고 해도, 터졌을때 대처를 할 수는 있어야 한다.  
다행히도 AWS는 스팟을 터뜨리기 전에 터뜨릴거라고 알림을 보내준다.  

이런식으로 말이다.  

```javascript
{
  "version": "0",
  "id": "a99d3f53-4a7c-4153-a1a5-48957fc83b8f",
  "detail-type": "ECS Task State Change",
  "source": "aws.ecs",
  "account": "1111222233334444",
  "resources": [
    "arn:aws:ecs:ap-southeast-2:1111222233334444:task/4be29e5b-b05c-42a2-a596-be62090eea9b"
  ],
  "detail": {
    "clusterArn": "arn:aws:ecs:ap-southeast-2:1111222233334444:cluster/default",
    "createdAt": "2022-02-25T10:13:08.455Z",
    "desiredStatus": "STOPPED",
    "lastStatus": "RUNNING",
    "stoppedReason": "Your Spot Task was interrupted.",
    "stopCode": "SpotInterruption",
    "taskArn": "arn:aws:ecs:ap-southeast-2:1111222233334444:task/4be29e5b-b05c-42a2-a596-be62090eea9bEXAMPLE",
    ...
  }
}
```

그래서 저걸 트리거하고 싶다면 eventbridge rule을 만들어서  

![](/images/img_acb166aeb0d8aa5f.png)

이벤트를 이런식으로 넣어준다.  

```javascript
{
  "source": [
    "aws.ecs"
  ],
  "detail-type": [
    "ECS Task State Change"
  ],
  "detail": {
    "stopCode": [
      "SpotInterruption"
    ],
    "clusterArn": [
      "arn:aws:ecs:리전:계정번호:cluster/클러스터명"
    ]
  }
}
```
그리고 필요에 따라 SNS 같은걸 붙여서 이메일로 확인할 수 있게 하면 된다.
