## 기본 설정

디펜던시는 3개쯤은 기본으로 갖고간다 보면 된다.  

```javascript
futures = "0.3.30"
sea-orm = { version = "^0.12.0", features = [ "sqlx-postgres", "runtime-tokio-native-tls", "macros" ] }
tokio = { version = "1.35.1", features = ["full"] }
```

## 엔티티 만들기

seaorm은 엔티티를 기반으로 모든 데이터를 관리하는 것을 지향한다.  

근데 엔티티 정의는 다른 라이브러리들에 비해 좀 장황한 편이다.  
그냥 간단한 엔티티 하나만 정의해도  

```javascript
use sea_orm::entity::prelude::*;
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "student")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub name: String,
    #[sea_orm(column_name = "schoolName")]
    pub school_name: String,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {}

impl ActiveModelBehavior for ActiveModel {}
```
이정도는 기본으로 깔고가야한다.  
그리고 저 구조체들이 매크로 생성을 통해 기묘하게 결합되어있는 형태라서, 저 세트를 각각 별도의 파일이나 모듈 단위로 격리하는 것을 권장하는 것 같다.  

난 일단 원코드로 가겠다.

## CRUD 돌려보기

seaorm에서 데이터를 어떤 형태로 관리하는지 한번 훑어보자.  

데이터의 생성은 이런식으로 할 수 있다.  

![](/images/img_549f035de127b8bc.png)
객체를 생성하고, active 모델로 전환한 다음에 insert 명령을 실행하면 된다.  

<br>

![](/images/img_f50fb8859f5d65cd.png)
그럼 이렇게 들어갈 것이다.  

수정은 그에 비해 좀 간편한 편이다.

저기서 model을 생성할때는, primary key를 기준으로 찾기 때문에 다른건 일치하지 않아도 된다.
