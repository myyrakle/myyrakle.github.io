k8s 위에서 동작하는 만들게 되면, 이런저런 리소스와 파라미터들을 다양하게 넣어야 하는 일이 많다.  
helm은 그런걸 패키지 단위로 뭉쳐서 다룰 수 있게 해준다.  

helm은 중앙화된 레지스트리가 따로 있진 않다. 프로토콜만 제공할 뿐이고, 각자 알아서 서버를 잘 띄워야 한다.  
정적 서버만 있으면 돼서 무료로 배포할 방법은 많지만, helm 패키지를 배포하는 가장 간단하고 저렴한 방법은 대개 Github Page를 사용하는 것이다.

<br>

<br>

## helm chart 구성

helm 문법과 구조를 따라서 helm chart를 구성한다.  

![](/images/img_09adb197cab912a0.png)
차트 정의와  

<br>

![](/images/img_ac41e3af9f586190.png)
파라미터 구성을 위한 values.yaml  

<br>

![](/images/img_de58be03ce6d0c1d.png)
그리고 helm install시에 apply로 설치할 쿠버 리소스 정의를 넣어주면 된다.  

준비가 되었다면 docs라는 폴더를 만들고, 거기에 helm의 결과물을 말아준다.  

```javascript
mkdir -p docs
helm package helm -d docs/
```

![](/images/img_e5782e40e6c6a86f.png)

![](/images/img_fc7dff771bf51631.png)
그럼 이런 식으로 압축파일과 index.yaml 하나가 생성될 것이다.  
이게 helm의 표준 형식이다. 이 경로로 저 파일 2개가 서빙되면 되는거다.  

생성한 뒤에는 commit & push해서 올라가게 만든다.  
그리고 레포지토리에서 Page를 활성화하되, 경로를 docs로 잡아준다.  

![](/images/img_da7967f59c069786.png)

![](/images/img_6a0e02bf64ef543b.png)
그래서 이렇게 공개가 되면 준비는 다 끝난 것이다.  

이후에는 저 경로로 helm 패키지를 추가하고 파라미터를 설정해서 install을 날리면 된다.  

```javascript
helm repo add bellboy https://myyrakle.github.io/bellboy/
helm install bellboy bellboy/bellboy ...
```
