cgroup은 리눅스 커널의 주요 기능 중 하나다. control group의 축약이다.  
특정 계정 단위로 CPU/메모리 사용량, 디스크 i/o 사용량, 네트워크 트래픽 사용량, 프로세스 수 등을 제한할 수 있게 해준다.  

컨테이너 기술의 핵심을 이루고 있기 때문에, 서버-인프라와 관계된 작업을 한다면 거의 기본 소양에 가까운 녀석이기도 하다.  
Docker/k8s 등에서 지원하는 컨테이너 간 리소스 격리 기능은 전부 cgroup 기능을 그대로 갖다 쓴 거라고 보면 된다. 사실 얘들이 직접 깎은건 별로 없다. 있는걸 잘 조합했을 뿐이다.  

<br>

<br>

<br>

## 버전 v1, v2

cgroup은 현재 2개의 버전으로 파편화되어있다.  
다음 명령을 사용하면 현재 리눅스 커널이 어떤 cgroup을 지원하는지 확인할 수 있다.  

```javascript
mount | grep cgroup
```

![](/images/img_1bc26d6129b8b770.png)
이러면 cgroup v2를 쓰는 것이다.  

v1이라고 해서 기본적인 기능이 부족하거나 버그가 많거나 한건 아니다. 복잡한거 좀 정리하고 인터페이스 수준의 안정화를 완료한 것이 cgroup v2다.  

커널 버전 4.5부터는 v2를 지원한다.  
하지만 v1도 아직 레거시에서 많이 쓰이고 있고, 큰 문제는 없다. 오히려 2008년부터 오랫동안 써왔기 때문에 어떤 측면에서는 더 안정적이라고 할 수도 있다.

## cgroup 사용해보기 - 메모리 사용 제한 (v2)

한번 이게 어떻게 동작하는지를 직접 돌려보면서 확인해보자.  

가장 먼저 /sys/fs/cgroup이라는 경로에 새로운 cgroup 디렉터리를 만들어줘야 한다.  

```javascript
sudo mkdir -p /sys/fs/cgroup/test_demo
```

![](/images/img_ca08bc67c85bf5e2.png)
이건 매우 특수한 경로라서, 빈 폴더만 만들어도  

<br>

![](/images/img_aca4d8f0f3a07b65.png)
바로 이렇게 자동으로 뭘 채워준다.  
이 각각의 파일이 cgroup 제어용 설정 파일이다.  

<br>

![](/images/img_37198c0dd274abb6.png)
실제로 보면 이런 식으로 설정값이 들어있다.  

여기서 우리는 가장 티가 잘 나고 확인하기 쉬운 "메모리 제한"을 걸어보겠다.  
메모리 제한에 도달하면 OOM으로 죽게 하는 것이다.  

다음 명령은 cgroup의 메모리 최대치를 100MB로 설정한다.  

```javascript
echo "100M" | sudo tee /sys/fs/cgroup/test_demo/memory.max
```

![](/images/img_b4a2d21aa6c4e9b0.png)

메모리 스왑 설정도 꺼야 한다. 안그러면 메모리를 초과해도 메모리 스왑을 사용하고 OOM을 당하지 않는다.  

![](/images/img_af88c80b476a2038.png)
이렇게 max로 설정되어있으면 스왑을 무제한으로 쓰는 것이다. 저거 다 쓸때까지 OOM KILL을 당하지 않는다.  

<br>

```javascript
echo "0" | sudo tee /sys/fs/cgroup/test_demo/memory.swap.max
```

![](/images/img_bc74a88cd5111483.png)
스왑을 끄고 싶다면 이렇게 끄면 된다.  

그 다음에 할 것은, cgroup에 프로세스를 등록하는 것이다. 방법은 꽤 쉽다.  

```javascript
cat /sys/fs/cgroup/test_demo/cgroup.procs
```

![](/images/img_3940fa112e590f22.png)
cgroup.procs라는 파일에 프로세스의 PID를 넣어주면 된다.  
그래서 프로세스를 실행해서 PID를 받아온 다음에 넣어줘야만 한다. 프로세스를 시작하기 전에는 cgroup에 할당할 수가 없다.  

파이썬으로 메모리를 천천히 점유하는 코드를 짜서 실행해봤다.  

```javascript
# 메모리를 10M씩 천천히 계속 늘려가며 점유하는 프로그램

import time
import os

memory = []

print("Process ID :", os.getpid())

# 엔터 대기...
input("Press Enter to start allocating memory...")

while True:
    # bytearray를 생성하고 실제로 메모리에 쓰기
    chunk = bytearray(10 * 1024 * 1024)
    # 메모리를 실제로 사용하도록 강제 (0xFF로 채우기)
    for i in range(0, len(chunk), 4096):  # 페이지 단위로 쓰기
        chunk[i] = 0xFF
    memory.append(chunk)
    time.sleep(3)  # 3초 대기
    print(f"Allocated memory: {len(memory) * 10} MB")
```

![](/images/img_5c57838f93ced11f.png)
실행하면 PID를 출력한 채로 멈추게 했다.  

저걸 이렇게 집어넣으면 된다.  

```javascript
echo [PID] | sudo tee /sys/fs/cgroup/test_demo/cgroup.procs
```

![](/images/img_0e5839b49392d1a5.png)

![](/images/img_db9ce1eb5b6702cf.png)

그리고 그 상태에서 실행을 재개하면  

![](/images/img_4ba1b72fec93c352.png)
100MB을 먹은 상태에서 터질 것이다.  
이게 cgroup이 해주는 것이다.  

Docker가 하는 것도 내부적으로 이렇게 파일 조작하고 하는 것이다.
