AWS의 람다는 정해진 런타임 이외에도 다양한 언어를 사용할 수 있게끔 커스텀 환경을 제공한다.  

일단 함수를 생성할 때 사용자 지정 런타임으로 설정해서 생성해야 한다.  

![](/images/img_19475668b9d69f8a.png)
1이든 2든 둘다 잘 되는 것 같다.  

사용자 지정 런타임이 동작하는 원리는 그닥 복잡하진 않다.  

![](/images/img_3d6c3cd956dfd4f2.png)
그냥 bootstrap이라는 실행파일이 안에 있으면 그걸 실행시켜준다.  
당연히 실행파일은 람다 규격에 맞게 통신을 수행할 수 있어야 하고...  

이제 저기 올릴 바이너리 파일을 빌드해보자.  
빌드는 되도록 리눅스에서 하는걸 권한다. 윈도우즈에서는 빌드시 링커 관련해서 문제가 있다.  

먼저 프로젝트를 하나 파고  

![](/images/img_75d3b5eea242b4d8.png)

리눅스 타겟을 추가한다.  
람다는 내부적으로 리눅스로 동작해서, 바이너리도 거기에 맞춰서 뿜어야 하기 때문이다.  

```javascript
rustup target add x86_64-unknown-linux-musl
```

![](/images/img_8c25aca5de2c03a7.png)

cargo.toml은 이렇게 했다.  

```javascript
[package]
name = "lambda_start"
version = "0.1.0"
authors = ["myyrakle <sssang97@naver.com>"]
edition = "2018"
autobins = false

[dependencies]
lambda_runtime = "^0.2"
serde = "^1"
serde_json = "^1"
serde_derive = "^1"
log = "^0.4"
simple_logger = "^1"
simple-error = "0.2.2"

[[bin]]
name = "bootstrap"
path = "src/main.rs"
```
[[bin]] 속성을 사용해 출력파일명이 boostrap이 되도록 지정했다.  

소스는 이렇게 짜면 된다.  

```javascript
use std::error::Error;

use lambda_runtime::{error::HandlerError, lambda, Context};
use log::{self, error};
use serde_derive::{Deserialize, Serialize};
use simple_error::bail;
use simple_logger;

#[derive(Deserialize)]
struct CustomEvent {
    #[serde(rename = "firstName")]
    first_name: String,
}

#[derive(Serialize)]
struct CustomOutput {
    message: String,
}

fn main() -> Result<(), Box<dyn Error>> {
    simple_logger::init_with_level(log::Level::Debug)?;
    lambda!(my_handler);

    Ok(())
}

fn my_handler(e: CustomEvent, c: Context) -> Result<CustomOutput, HandlerError> {
    if e.first_name == "" {
        error!("Empty first name in request {}", c.aws_request_id);
        bail!("Empty first name");
    }

    Ok(CustomOutput {
        message: format!("Hello, {}!", e.first_name),
    })
}
```
그냥 firstName이란 값을 전달받아서 출력하는게 다다.  

[https://github.com/myyrakle/aws-lambda-rust](https://github.com/myyrakle/aws-lambda-rust)  
만들고 복붙하는게 귀찮다면 이걸 클론받아서 돌려도 된다.  

빌드는 이런식으로 하면 된다.  

```javascript
cargo build --release --target x86_64-unknown-linux-musl
```

![](/images/img_2e5b070eb2d89357.png)

그러면 저기에 boostrap이란 이름으로 바이너리 파일이 생성될 것이다.  

![](/images/img_c8e7afc84081e788.png)

저걸 zip으로 잘 압축한다.  

```javascript
zip -j rust.zip ./target/x86_64-unknown-linux-musl/release/bootstrap
```
