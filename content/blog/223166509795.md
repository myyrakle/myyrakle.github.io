mockery는 mock 자동화 보조도구다.  
gomock과 라이벌인거같다.  

<br>

<br>

<br>

## 설치

먼저 cli를 설치해줘야 한다.  

```javascript
go install github.com/vektra/mockery/v2@v2.32.0
```

![](/images/img_95f9c9526cca3258.png)

![](/images/img_ee6eff8b643d0be6.png)
이렇게 실행이 되면 잘 깔린 것이다.  

<br>

<br>

<br>

## 예시

Redis와 같은 데이터베이스를 프로그램에서 연동하다보면 아래와 같이 작성할 수 있을 것이다.  

![](/images/img_3ed328bb2c3d2e59.png)
공통적인 인터페이스를 두고, 실제로 사용할 Real Redis 버전을 구현한다. (여기서는 진짜 Redis에 연동을 하지는 않았다.)  
이걸 인터페이스로 두는 이유는 Mock 객체를 다형적으로 사용할 수 있게 하기 위함이다.  

그럼 사용도 이런식으로 할 수 있다.  

![](/images/img_90b1e996162d1387.png)

![](/images/img_43146d516b8c6e5f.png)
잘 돌아간다.  
근데 메인 로직은 로직이고, 데이터베이스 같은 외부 요소에 대한 단위테스트를 하려면 준비를 또 해줘야 한다.  

<br>

<br>

<br>

## 방법 1: go generate 활용

go generate를 써서 Mock을 구현하고 싶은 인터페이스 위에다가 mockery cli 명령을 적절히 넣어줘서  

![](/images/img_4a7087c5e4739c22.png)

generate를 시킬 수 있다.  

![](/images/img_7d77e04640796cd0.png)
그럼 이렇게 소스코드가 생길 것이다.  

<br>

<br>

<br>

저런 mock 생성을 중앙화해서 체계적으로 관리하고 싶다면 그냥 yaml을 쓰는 편이 깔끔하다.  
다음과 같은 형태로 루트에다 yaml을 작성해주고  

![](/images/img_047f3df2adc5add7.png)

![](/images/img_c96a2977a9524c89.png)

## Mock 살펴보기

Mock 생성물이라 해봐야 사실 별거 없다.  
원본 인터페이스의 각 메서드에 대해서 empty 구현을 두고, 테스트 시점에 주입할 수 있도록 하는 것이다.  

그래서 이런 의미없는 보일러플레이트들만 잔뜩 있다.  

![](/images/img_e35ffbc41652d2f1.png)
이걸 직접 분석하거나 관리할 일은 없다. mockery cli로 싱크만 계속 맞춰주면 된다.  

<br>

<br>

<br>

## 테스트 작성하기

그럼 저 mock 생성물을 이용해서 간단한 단위테스트를 작성해보겠다.  
mock 객체를 생성하고, On으로 각 메서드가 어떤 조건으로 호출되었을때 어떤 값이 리턴되도록 할지를 정의하면 된다.  

![](/images/img_483b6720fa4a5d10.png)
그 이후에는 그냥 go test 내장기능을 이용해서 테스트를 수행하면 된다.  

<br>

![](/images/img_1abb41285258e38e.png)
잘 된다.
