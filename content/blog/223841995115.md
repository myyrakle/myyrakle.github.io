Collection은 qdrant에서 가장 기본이 되는 단위다.  
일반적인 데이터베이스들의 테이블, Elasticsearch의 Index에 대응된다.  

상단의 개발자 콘솔을 사용하면 손쉽게 데이터를 다뤄볼 수 있다.  

![](/images/img_941317b4f5ee1e86.png)

![](/images/img_5fb9df063db7f3dd.png)
이걸 통해서 간단한 실습을 해보겠다.  
/collections API는 컬렉션 목록을 조회하는 API다. 아직 아무것도 만든적이 없어서 들어있는건 없다.

## 컬렉션 생성

컬렉션을 생성하는 것은 그리 어렵지 않다.  
다음은 256짜리 Dot Product용 벡터 컬렉션 "products"를 만드는 간단한 요청이다.  

```javascript
PUT /collections/products
{
    "vectors": {
      "size": 256,
      "distance": "Dot"
    }
}
```

## 멀티 벡터

위에서 만든 벡터 컬렉션은 단일 벡터만을 가지는 컬렉션이다.  
하지만 벡터 필드를 여러개 뚫는 것도 가능하다.  
vectors 안에 또 키를 여러개 둬서 깔면 된다.  

```javascript
PUT /collections/products2
{
    "vectors": {
      "image": {
        "size": 256,
        "distance": "Dot"
      }, 
      "text": {
        "size": 256,
        "distance": "Dot"
      }
    }
}
```

## 컬렉션 alias

alias는 컬렉션에 대해 붙일 수 있는 또 다른 별명이다.  

Elasticsearch에서는 인덱스를 새로운 인덱스로 옮길때, 다운타임 없이 교체하기 위해서 alias 기반 스위칭을 사용하곤 한다. qdrant도 같은 방법을 지원한다.  

alias의 생성은 이런 식으로 이뤄진다.  

![](/images/img_5a41df96c18029ad.png)
그냥 컬렉션에 products_prod란 이름을 붙인 것이다.  

그럼 이제 products_prod란 이름으로도 products 컬렉션을 사용할 수 있게 된다.  

만들어진 alias 목록은 전용 api로 확인할 수 있다.  

![](/images/img_34e153e7abd515cc.png)

alias의 삭제와 생성을 단일 BATCH로 묶어서 쏘면 다운타임 없이 alias가 다른 컬렉션을 가리키게 할 수도 있다.  

![](/images/img_e5e8f14735ba2b12.png)
운영에 있어 거의 필수적인 트릭이다.
