엘라스틱캐시는 인메모리 NoSQL 시스템을 제공하는 서비스다.  
NoSQL 시스템은 Memcached와 Redis 둘중 하나를 선택할 수 있다.  

여기서는 일단 대중적인 Redis를 세팅하는 법만 간단하게 다뤄본다.  

들어가서 생성을 누른다.  

![](/images/img_e286671a4c6fc182.png)

그럼 레디스와 멤캐시를 고를 수 있는데, 취향껏 고르면 된다.  
나는 레디스다.  

![](/images/img_445cf738c9990d9d.png)

이름 짓고. 노드 유형을 잘 고른다.  

![](/images/img_c2f26fe3e199fc18.png)
기본값이 제일 비싼걸로 되어있기 때문에 대충 만들었다가는 비용 폭탄을 맞을 수 있다.  
적당히 낮은걸로 시작해서 나중에 필요할때 올리자  

<br>

![](/images/img_4c583c649d957832.png)
그리고 보안그룹을 잘 골라준다.  
이것도 꽤 중요하다.  
엘라스틱캐시는 정책상 '외부 액세스'를 원천적으로 차단한다.  
RDS의 경우는 (권장되진 않지만) 퍼블릭 액세스를 열어서 대충 쓸수도 있는데, 엘라스틱캐시는 외부 공개도 안되고, 같은 VPC 내에서의 접근만 허용한다.  
퍼블릭액세스란게 극단적인 성능 최적화에 걸림돌이 된다나 뭐라나...  

그렇게 해서 만들면 이런식으로 페이지가 뜨게 될 것이다.  

![](/images/img_2f81c265b4a73a81.png)

![](/images/img_a7b821db219f802f.png)
그리고 저 위에서 클러스터를 펼치면 '기본 엔드포인트'라는 항목으로 도메인이 뜨는데, 그걸로 접속할 수 있다.  
한번 해보자.  

EC2 인스턴스를 동일한 VPC 보안그룹에 만들어주고, 다음과 같이 redis 접속 도구를 설치한다.  

```javascript
sudo yum install -y gcc
wget http://download.redis.io/redis-stable.tar.gz && tar xvzf redis-stable.tar.gz && cd redis-stable && make
sudo cp src/redis-cli /usr/bin/
```

![](/images/img_630f51a342524496.png)

그리고 이렇게 접속을 시도해서 잘 되면 잘 된 것이다.  

![](/images/img_954d9229e21823cb.png)
