Thread들은 기본적으로 실행시간이 정해져있지 않다.  
만약 스레드 6개를 동시에 띄운다고 하면  

![](/images/img_070b3a0ad3a03a42.png)
이렇게 제멋대로 실행되고 제멋대로 종료될 것이다.  
대체로 이런 동작 자체에는 문제가 없으나, 구현 요구사항에 따라서 여러개의 스레드가 다 준비가 되었을때 동시에 시작이 되도록 하고 싶을 수도 있다.  

그럴때 사용하는 것이 Barrier라고 부르는 Lock 패턴이다.  
이건 Lock이 해제되는 임계 개수를 지정하고, 스레드들이 특정 코드 지점에 도달할때마다 lock을 건다.  
그리고 모든 스레드가 코드 지점에 도달하면 lock을 해제하고 거의 동시에 실행되도록 유도한다.  

![](/images/img_5df91f3a84fc073c.png)

아래는 Rust에서 barrier를 사용하는 예제 코드다.  
6개의 스레드를 띄워서 동시에 실행되도록 유도한다.  

```javascript
fn main() {
    use std::sync::{Arc, Barrier};
    use std::thread;

    let mut handles = Vec::with_capacity(6);

    // 크기 6짜리 배리어
    let barrier = Arc::new(Barrier::new(6));

    // 스레드 6개 발주
    for _ in 0..6 {
        let barrier = Arc::clone(&barrier);

        handles.push(thread::spawn(move || {
            println!("before wait");
            barrier.wait(); // 6개가 전부 여기에 도달할 때까지 기다림
            println!("after wait");
        }));
    }

    for handle in handles {
        handle.join().unwrap();
    }
}
```
