요즘은 시계열이 유행인가?  
짬뽕형 DB의 극한을 달리는 MongoDB답게 시계열 데이터에 대한 기능도 제공한다.  

기본기능으로 내장되어있고, 버전 5.0부터 사용가능하다.  
2021년에 출시된 기능이라 비교적 최신 기능이다.  

<br>

<br>

<br>

## 다른 시계열 DB들과의 비교

time-series 컬렉션이 지원되지 않을때는 당연히 다른 시계열 DB들과 비교하면 많이 구렸다.  

하지만 time-series가 지원되고 나서부터는 성능이나 사용성 면에서 꽤나 쓸만해진 것 같다.  
데이터 압축, 빠른 삽입, 데이터 만료 등의 기능을 제공한다.  

timescaleDB나 influxDB와 비교해서 더 좋다고 하기는 어렵지만, 기존에 MongoDB를 사용하고 있었다면 괜찮은 선택이 될 수 있을 것 같다.  

<br>

<br>

<br>

## 컬렉션 만들기

몽고는 원래 컬렉션을 명시적으로 만들지 않아도 되지만, 시계열 컬렉션은 명시를 해줘야 한다.  
생성 시점에 시계열에 대한 설정을 해줘야 하기 떄문이다.  

이런식이다.  

```javascript
db.createCollection(
    "weather",
    {
       timeseries: {
          timeField: "timestamp",
          metaField: "metadata",
          granularity: "seconds"
       }
    }
)
```

![](/images/img_deaa68909d1b5708.png)
timeFields는 실제로 시간값을 저장할 필드명이다.  
여기에 시간값을 넣으면, 몽고db가 그 시간값을 기준으로 적절히 최적화를 해준다.  

metaField는 그냥 메타데이터 저장용이고, granularity는 시간을 저장할때 어떻게 파티셔닝을 할건지, 같은걸 정하는 기준값이다. 보통은 초 단위로 하는게 일반적이고, 기본값이다.  
분리 단위가 초 단위라고는 해도 실제 저장은 밀리초 단위까지 할 수 있다.
