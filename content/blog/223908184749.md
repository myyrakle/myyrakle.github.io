wire는 구글에서 만든 DI 라이브러리다.  

fx 같은 기존의 DI 라이브러리들이 런타임 리플렉션 기반으로 DI를 처리했다면, 이건 codegen 기반으로 코드를 생성해서 되도록 컴파일타임에 처리하는 스타일이다.  

런타임 처리가 비교적 적어 성능적으로도 좀 낫고, 종속성 오류를 컴파일타임에 체크할 수 있다는게 강력한 장점이기도 하다.  

<br>

<br>

<br>

## 설치

이건 codegen 프로그램을 돌려서 파일을 생성하는 것이기 때문에, 전용 CLI 도구가 필수적이다.  

```javascript
go install github.com/google/wire/cmd/wire@latest
```

![](/images/img_c745196fda937791.png)

![](/images/img_8410de45dd5cb54a.png)
적당히 깔아서, 실행이 되면 된 것이다.  

라이브러리도 필요하다.  
일종의 매니페스트 설정을 하는데 쓴다.  

```javascript
go get github.com/google/wire
```

![](/images/img_95dd3d5a198ea7fa.png)

## wire 기본 세팅

먼저 wire.go라는 이름의 특수한 설정파일을 생성해야 한다.  
이건 딱 codegen용이고, 실제 컴파일 대상에는 속하지 않아야 하기 때문에 빌드에서 제외되도록 설정해준다.  

![](/images/img_82fd882195d3094d.png)
최종적으로 만들어지는 객체를 반환타입으로 삼고, 빈 값을 반환하도록만 해주면 된다. 어차피 타입만 필요하다.  
그리고 wire.Build 내에 들어가는 생성 함수들의 매개변수 타입과 반환타입을 분석해서 DI 사이클 코드를 빌드해주는 것이다.  

저 상태에서 wire CLI로, wire.go가 존재하는 경로를 찍어 돌려주면  

![](/images/img_7862341968f992ba.png)

wire_gen이라는 파일이 생긴다.  

![](/images/img_6748ef798edead94.png)

![](/images/img_4641b1d493fc8300.png)
이게 우리가 최종적으로 사용할 결과물이다.  

그냥 그대로 가져다가 적절히 쓰면 된다.  

![](/images/img_ea88d624488fffcc.png)

![](/images/img_8a8cdc4c80cf7576.png)

wire의 codegen 방식을 사용할때의 장점은, 성능도 성능이지만 DI 누락을 더 빠르게 발견할 수 있다는 것이 크다.  
예를 들어 저기서 필수적인 DatabaseAdapter의 생성이 누락된다면  

![](/images/img_a3333286ce989ba1.png)

![](/images/img_a9e6c5c7a5324d89.png)
wire를 실행하는 단계에서 즉시 알아챌 수 있다.  

또 미사용 종속성에 대해서도 민감하게 감지한다.  

![](/images/img_45b1c4cf052f4216.png)

![](/images/img_48850670dd9653a1.png)
이것도 오류로 처리한다.  

기본적인 원리나 사용법은 이 정도다.
