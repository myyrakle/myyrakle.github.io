egui는 Rust의 주요 GUI 환경 중 하나다.  

간단하고 가벼운 형태를 지향하며, 앱UI와 웹UI를 동시에 제공한다.  
이 포스트에서는 앱만 다룬다.  

이 녀석은 딱히 빠르거나 강력한 구조를 지향하고 만들어지지는 않았다. 렌더링에 대해서도 별다른 최적화를 하지는 않고 만든 것 같더라. 뭔가 이벤트가 발생하거나 상호작용이 발생하면 항상 전체 레이아웃을 렌더링한다.

## 설치

주요 종속성은 eframe이라는 녀석이다. 다음과 같이 종속성을 추가해준다.  

```javascript
[dependencies]
eframe = { version = "0.26.2", features = [
    "default",
    "__screenshot", # __screenshot is so we can dump a screenshot using EFRAME_SCREENSHOT_TO
] }
env_logger = { version = "0.10", default-features = false, features = [
    "auto-color",
    "humantime",
] }
```

## Hello World 예제

아래는 전체 코드다.  

```javascript
#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")] // hide console window on Windows in release

use eframe::egui;

fn main() -> Result<(), eframe::Error> {
    env_logger::init(); // Log to stderr (if you run with `RUST_LOG=debug`).

    let options = eframe::NativeOptions {
        viewport: egui::ViewportBuilder::default().with_inner_size([320.0, 240.0]),
        ..Default::default()
    };

    // Our application state:
    let mut name = "Arthur".to_owned();
    let mut age = 42;

    eframe::run_simple_native("My egui App", options, move |ctx, _frame| {
        egui::CentralPanel::default().show(ctx, |ui| {
            ui.heading("My egui Application");
            ui.horizontal(|ui| {
                let name_label = ui.label("Your name: ");
                ui.text_edit_singleline(&mut name)
                    .labelled_by(name_label.id);
            });
            ui.add(egui::Slider::new(&mut age, 0..=120).text("age"));
            if ui.button("Increment").clicked() {
                age += 1;
            }
            ui.label(format!("Hello '{name}', age {age}"));
        });
    })
}
```
붙여넣어서 우선 실행해보자.

## 구조 분석

막 그렇게 구조가 복잡하진 않다.  

![](/images/img_19bc4e02bd7f4c4a.png)
우선 기본 창 크기부터 정의한다.  

<br>

![](/images/img_2588980361f2ea0c.png)
그리고 run_simple_native를 호출해서 앱을 띄운다. 여기에서는 아까 정의한 옵션과 Panel 객체를 넣어서 창 정보를 정의한다.  

egui는 렌더링 최적화가 조금 모자라다고 했었다.  

![](/images/img_950421030dec957d.png)
이 코드를 애니메이션이나 User 상호작용이 있을때마다 항상 그린다.  
그래서 그걸 염두에 두고 잘 짜줘야 한다. 여기서 부하가 걸리면 안된다.  

아무튼 저기서 레이아웃 요소들을 하나하나 쌓아올리는 형태다. 정의한 순서대로 위-아래로 배치된다.  
1. heading으로 헤더 영역을 두고,  
2. horizonatal로 가로 영역을 두고 라벨과 텍스트 입력창을 만들었다.  
3. 옆으로 움직여서 값을 조절하는 슬라이더를 만들었다. 이를 통해 age 변수값이 수정되게 한다.  
4. button을 추가하고, 그게 눌렸을때마다 age 값을 1 늘리게 한다.  
5. 텍스트 label을 하나 추가한다.  

다른 gui 프레임워크들에 비하면 구조가 꽤 이질적인 면도 있다. 특히 이벤트 핸들링이 콜백을 통해 구성되지 않는다는 점이 특이하다. 코드의 복잡성을 줄이기 위함이라나.  

아무튼 이벤트 핸들링도 프레임 한번 돌때마다 조건문을 다는 식으로 구성이 된다. 저기 버튼 핸들링 처럼 말이다.  
라이프타임 문제가 발생할 일이 적으니 쓰기 쉬운 부분은 확실히 있는 것 같다.
