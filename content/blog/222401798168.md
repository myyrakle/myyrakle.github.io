![](/images/img_b9a704215516e5e0.png)
AWS Personalize는 개인화에 최적화된 기능을 제공하는 머신러닝 서비스다.  

가장 대표적인 사용례를 든다면, 추천상품 목록을 띄워주는 것이 있겠다.  
사용자가 어떤 상품을 보고, 찜하고, 구입했는지를 수집해서 그 상품들과 관련있는 상품들을 보여주는 것이다.  

상당히 단순하게 정형화되어있어서, 머신러닝 지식 없이도 큰 어려움 없이 사용할 수 있다는 장점이 있다.  

다만, 실시간으로 학습데이터를 변동시킬 일이 많다면, 비용이 상당히 깨질 수 있다.  
내 경우에는 매일마다 수십만개의 상품 데이터를 계속 갱신시키는 작업을 했더니, 한달이 700 달러 가까이가 깨졌다.

## 데이터 준비 및 밑작업

테스트에 사용할 데이터는 여기서 받아보자  
[http://files.grouplens.org/datasets/movielens/ml-latest-small.zip](http://files.grouplens.org/datasets/movielens/ml-latest-small.zip)  

아래의 4가지 csv 파일이 존재한다.  

![](/images/img_11b4c2863aa804b3.png)

여기서 rating은 아래와 같이 바꿔주자.  
어떤 사용자가 어떤 아이템(영화)를 봤는지를 기록한 상호작용 데이터다.  

![](/images/img_8baf1571e841430f.png)

movies로 바꿔주겠다.  
실제 영화에 대한 데이터 목록이다.  

![](/images/img_477b8a9f1981a64a.png)

그리고 S3에 업로드한다.  

![](/images/img_12439786af8df468.png)

그리고 퍼스널라이즈에서 S3에 접근할 수 있게, 저 버킷에 권한을 추가해준다.  

![](/images/img_2854c2b8f37a87cf.png)
저기서 버킷 정책에  

이런 식으로 넣어주면 된다.  

```javascript
{
    "Version": "2012-10-17",
    "Id": "PersonalizeS3BucketAccessPolicy",
    "Statement": [
        {
            "Sid": "PersonalizeS3BucketAccessPolicy",
            "Effect": "Allow",
            "Principal": {
                "Service": "personalize.amazonaws.com"
            },
            "Action": [
                "s3:GetObject",
                "s3:ListBucket"
            ],
            "Resource": [
                "arn:aws:s3:::버킷명",
                "arn:aws:s3:::버킷명/*"
            ]
        }
    ]
}
```
Personalize에 권한을 주는 방식이 아니라 S3에서 열어주는 형식이란 점에 유의한다.  
그 반대는 안된다.

## 데이터그룹 생성

이건 학습에 필요한 초기 데이터를 넣어주는 부분이다.  
한번 넣고 나면 다시 사용하진 않는다.  

이제 Personalize에 들어가 데이터 그룹 생성을 클릭한다.  

![](/images/img_d32c2f316c38d422.png)

그럼 대충 저렇게 뜰 것인데,  

![](/images/img_6d2e3b7b94800848.png)
이름을 잘 짓고 스키마를 정의해준다.  
여기에는 아까의 rating을 넣어줄 것이라서, fields에 RATING만 추가해주면 된다.  

그러면 데이터 그룹이 생성되고 다음과 같은 대시보드 페이지가 뜰 것인데  

![](/images/img_e9afab27c3458678.png)
User-Item Interfaction 데이터를 먼저 임포트해줘야 한다.  
말 그대로 "어떤 사용자가 어떤 상품에 관심을 가졌는지" 등에 대한 관계를 정의하는 데이터다.  

아까 RATING에 맞춰서 이렇게 해주면 된다.  

![](/images/img_8baf1571e841430f.png)

![](/images/img_21ff8e73114fa451.png)
여기서 USER_ID와 ITEM_ID, TIMESTAMP는 필수 필드다.  
안넣으면 백만년동안 안된다고 튕길 것이다.  

그리고 실행역할과 읽어올 csv 파일을 잘 지정해준다.  

![](/images/img_4d558831dd222518.png)

그럼 이제 데이터셋을 생성할 것인데, 시간이 조금 걸릴 수 있다.  

![](/images/img_dd19a115ae9b9828.png)

그럼 이제 솔루션을 생성할 수 있게 된다.  

![](/images/img_a69dd6e1e6a8f381.png)
