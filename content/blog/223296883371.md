CDC(Change Data Capture:변경 데이터 캡쳐)는 DB 상에서 데이터가 변경될때마다 그걸 가져다 뭔가 액션을 취하는 기능 요소를 말한다.  
변경을 기반으로 다른 데이터소스에 데이터를 동기화하거나, 웹훅을 쏜다거나 할때 자주 사용되는 패턴이다.  

이걸 구현할 때는 보통 Kafka 같은걸로 데이터 폴링하면서 쏘기도 하고 그러는데, RDS AuroraDB를 사용할 경우에는 이런 목표를 더욱 간단하게 달성할 수 있다. MySQL, PostgreSQL 둘 다 된다.  

<br>

![](/images/img_a5e789fbd0e442ec.png)
RDS AuroraDB는 자체적으로 위의 그림과 같이 DB에서 AWS Lambda를 바로 호출할 수 있는 기능을 제공한다. 이걸 응용해서 테이블에 update 트리거를 걸고, 트리거 프로시저에서 Lambda를 호출하면 Lambda가 그걸 받아서 추가적인 처리를 하도록 할 수 있다.  

<br>

<br>

<br>

## Lambda 확장 설치

전용 확장을 설치해줘야 한다.  
아무데나 cli 들어가서 아래 커맨드를 입력해준다,  

```javascript
CREATE EXTENSION IF NOT EXISTS aws_lambda CASCADE;
```

## IAM 권한 설정

먼저 RDS가 Lambda에 접근할 수 있게 해줄 권한과 정책을 설정해줘야 한다.  

먼저 역할을 하나 만든다.  

![](/images/img_40f2ad08df343084.png)

```javascript
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Service": "rds.amazonaws.com"
            },
            "Action": "sts:AssumeRole"
        }
    ]
}
```
그리고 위의 신뢰 정책을 복붙해 넣어준다.  

<br>

![](/images/img_2cc8118fb2b8270e.png)
이름은 적당히 짓고  

<br>

```javascript
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "lambda:InvokeFunction",
            "Resource": "*"
        }
    ]
}
```

![](/images/img_33395131a80b79c9.png)
역할에 권한을 추가해준다.  
나는 인라인 정책으로 넣었다.

## RDS에서 Lambda 호출해보기

함수의 형태로 호출이 가능하다.  
아래는 방금 만든 Lambda 함수를 RDS에서 호출하는 간단한 예제다.  

```javascript
SELECT * from aws_lambda.invoke(aws_commons.create_lambda_function_arn('cdc_function', 'ap-northeast-2'), '{"body": "Hello from Postgres!"}'::json );
```

## 트리거로 CDC 구성하기

이번에는 트리거를 활용해서 Lambda를 통한 CDC를 구성해보겠다.  

먼저 간단한 테이블을 하나 생성했다.  

![](/images/img_14036b9fbf5a1d5e.png)
별건 없다.  

그리고 아래와 같은 형태로 함수와 트리거를 생성해준다.  

```javascript
CREATE OR REPLACE FUNCTION public.trigger_test_table_function()
 RETURNS trigger
 LANGUAGE plpgsql
AS $function$
    begin
        IF (TG_OP = 'DELETE') THEN
            perform aws_lambda.invoke(
           		aws_commons.create_lambda_function_arn('cdc_function', 'ap-northeast-2'), 
           		json_build_object(
           			'type', 'delete',
           			'value', row_to_json(old)
           		)
           	);
        ELSIF (TG_OP = 'UPDATE') THEN
            perform aws_lambda.invoke(
           		aws_commons.create_lambda_function_arn('cdc_function', 'ap-northeast-2'), 
           		json_build_object(
           			'type', 'update',
           			'value', row_to_json(new)
           		)
           	);
        ELSIF (TG_OP = 'INSERT') THEN
            perform aws_lambda.invoke(
           		aws_commons.create_lambda_function_arn('cdc_function', 'ap-northeast-2'), 
           		json_build_object(
           			'type', 'insert',
           			'value', row_to_json(new)
           		)
           	);
        END IF;
        RETURN NULL; 
    END;
$function$;

CREATE TRIGGER trigger_test_table
AFTER INSERT OR update or delete
ON test
FOR EACH ROW
EXECUTE Function trigger_test_table_function();
```
어떤 operation으로 날라온건지, 그리고 테이블 row의 전체 값을 그대로 쏴주도록 했다.
