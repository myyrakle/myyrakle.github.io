Docker에서 로그를 조회하는 방법과, 저장을 관리하는 방법 등을 정리해본다.  

<br>

<br>

<br>

## 도커 로그 조회

기본적으로 로그는 logs 명령을 통해서 조회할 수 있다.  

```javascript
docker logs 컨테이너ID
```

![](/images/img_c733c27ca8649c0e.png)

끝에서 10줄만 보고 싶다면 tail 플래그를 주면 되고  

![](/images/img_da089816368b94e0.png)

실시간으로 쌓이는 로그를 tailing하려면 -f 플래그를 주면 된다.  

![](/images/img_941a84c035f555f8.png)

## 로그 파일의 위치

로그 파일의 위치는 설치 환경에 따라 다를 수 있다. 정확한 위치를 확인하는 방법은, 컨테이너 정보를 직접 까보는 것이다.  
다음 명령어로 경로를 찾고  

```javascript
sudo docker inspect 컨테이너ID --format "{{.LogPath}}"
```

![](/images/img_1fe0b68eaa01cd87.png)

그 경로를 까보면 로그파일과 잡다한 메타데이터 계열의 파일들이 함께 들어있고  

![](/images/img_f3e7d784ff1807df.png)

<br>

![](/images/img_bb2f2cb28f4d1a16.png)
로그도 이런 식으로 저장이 되어있을 것이다.

## Log Driver 관리

docker의 기본 로그시스템에는 꽤 큰 문제가 있는데, 아무런 관리를 해주지 않는다는 것이다.  

retention이나 rotation 같은 기본적인 설정도 없기 때문에, 컨테이너에서 쌓는 로그는 언제나 append-only로 쌓이기만 한다. 컨테이너가 종료된다고 해서 로그를 정리하지도 않는다.  

그래서 직접 Docker를 제어해서 시스템을 굴릴 경우에는 로그 driver 설정을 건드려줄 필요가 어느 정도 있다.  

<br>

![](/images/img_eb4cd7ef43d26c01.png)
일단, docker는 기본적으로 json-file이라는 로그 드라이버를 쓴다.  
이건 rotation 같은 기능도 없고, 리텐션으로 자동 정리하거나 하는 기능도 없는 구린 드라이버다.  
이게 기본값인 이유는 딱히 이게 좋아서가 아니라, 단순히 하위호환성 같은 것 때문이다.  

디스크 관리를 효율적으로 하고 싶다면 local 드라이버를 쓰는게 권장된다.  
전역 설정을 바꾸는 방법은 다음과 같다.  
먼저 daemon.json 파일의 위치를 찾는다.  

```javascript
ps aux | grep dockerd
```

![](/images/img_23ed1332653e2dab.png)
그럼 저 config-file을 열어서  

<br>

![](/images/img_aa72e45203f382da.png)

```javascript
{
  "log-driver": "local",
  "log-opts": {
    "max-size": "100m"
  }
}
```
값을 적당히 이런 식으로 수정해준다.  
이러면 컨테이너마다 최대 100mb만큼의 로그를 보관하고, 그 이상은 버리겠다는 뜻이 된다.  

그리고 docker를 적당히 재시작하면  

```javascript
sudo systemctl restart snap.docker.dockerd
```

![](/images/img_c3898a01f83bae2c.png)
이제부터는 기본 드라이버가 local이 되고, 로컬 드라이브에 의해서 로그 정리가 어느 정도 될 것이다.  

글로벌 설정을 건드리고 싶지 않다면 그냥 컨테이너를 실행할때 직접 옵션을 넣는 방법도 있다.  

```javascript
sudo docker run -d --log-driver local --log-opt max-size=50m ubuntu:latest echo hello,world!
```

![](/images/img_28581f6258a9d1a5.png)
이러면 그냥 로컬 드라이버에, 50mb 크기 제한을 가진 컨테이너가 만들어진다.  

그리고 docker compose를 사용할 때도 logging 필드를 통해 log driver 설정을 할 수 있다.  

![](/images/img_1cd8edfdd3cf018c.png)
이런 식으로 말이다.
