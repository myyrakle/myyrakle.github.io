![](/images/img_1d5fbc95bf87bf99.jpg)
helm은 쿠버네티스 환경에서 주로 사용하는 일종의 프로젝트 매니저다.  

쿠버네티스 배포에 사용되는 yaml 코드나 설정을 템플릿으로 구성해서 재사용할 수 있게 해주고, 여러개의 controller 단위를 묶어서 관리할 수 있게 해준다.  
그리고 도커처럼 remote 레지스트리가 있고, push/pull이 가능하다. 히스토리 관리가 된다.  

내가 사용한 테스트 환경은 GKE다.  
온프레미스 환경일 경우 로드밸런서 설정은 동작하지 않을 것이다.

## 설치

linux나 mac 같은 unix 계열에서는 다음과 같이 설치할 수 있다.  

```javascript
curl -fsSL -o get_helm.sh https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3
chmod 700 get_helm.sh
./get_helm.sh
```

![](/images/img_ad6e6de777310e92.png)

설치되면 이렇게 CLI를 실행할 수 있어야 한다.  

![](/images/img_abbf2d9ddb8906ff.png)

windows의 경우에는 바이너리를 직접 다운받거나. choco 같은걸 써야한다.  
[https://github.com/helm/helm/releases](https://github.com/helm/helm/releases)

## 차트 만들기

helm은 chart라는 단위로 프로젝트를 관리한다.  
차트 안에는 여러개의 쿠버 yaml 파일과, 그 yaml을 보완하는 템플릿(tpl)들로 구성될 수 있다.  

프로젝트, 차트 생성은 다음과 같이 할 수 있다.  

```javascript
helm create 차트명
```

![](/images/img_aa6dc4cf1e911136.png)

그럼 이런식으로 프로젝트가 구성된다.  

![](/images/img_e3285a93fc5142d3.png)
루트경로에 있는 Chart.yaml은 해당 차트 자체에 대한 메타 정보를 정의하고, values는 여러가지 상수값들을 포함한다.  
templates/*.yaml들은 쿠버네티스 배포용 yaml 파일들이다.  

일단 날리고 처음부터 하나씩 해보자.  

```javascript
rm -rf templates/*
```

## 서버 띄우기

차트를 이용해서 한번 서버를 띄워보자.  
아래는 내가 말아놓은 node.js 기반 서버를 띄우는 간단한 구성이다.  

```javascript
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ .Release.Name }}-deploy
spec:
  replicas: 3
  selector: 
    matchLabels: 
      app: test-server
  template: 
    metadata: 
      labels: 
        app: test-server
    spec: 
      containers: 
        - name: test-server
          image: myyrakle/node-server-for-test
```

![](/images/img_2f10c6a89458154b.png)
다른거라면 {{}} 중괄호 표현식으로 특수한 값을 메타데이터에 집어넣었다는 것 정도다.  
저 값은 배포시에 전달되는 값이다.  
저 이름을 기반으로 레이블명같은걸 지정해서 충돌하지 않게 하는 용도다. 그래서 만약 실제 프로덕션 배포를 한다면 저 예제에서 labels 설정에도 붙여주는게 좋겠다.

## 배포하기

초기 배포의 경우에는 다음과 같은 형태로 수행할 수 있다.  

```javascript
helm install 릴리즈명 차트경로
```

![](/images/img_f004f5f208f645ba.png)
그럼 차트 단위로 리소스가 뜰 것이다.  

helm list 명령으로 배포된 전체 차트 리소스 목록을 조회할 수 있고  

![](/images/img_f659e465a8287517.png)

status로 간략 조회  

![](/images/img_e7d6b40d884e7938.png)

get all로 상세 조회가 가능하다.  

![](/images/img_66ea0023b8025182.png)

## 로드밸런서 추가해보기

위에서 만들었던 예제는 서버 인스턴스만 덜렁 있는거라서, 뭔가 잘 돌아가는지 확인이 어려웠다.  
이번에는 저 앞에 달아줄 로드밸런서를 정의하고, 이걸로 엔드포인트를 열여주자.  

```javascript
apiVersion: v1
kind: Service
metadata:
  name: {{ .Release.Name }}-lb-service
spec:
  selector:
    app: test-server
  ports:
    - protocol: TCP
      port: 80
  type: LoadBalancer
```

![](/images/img_d37908a11fe5f91d.png)
templates에 추가하면 된다.  
저 경로 잔뜩 늘어놓기만 하면 알아서 읽어서 추가해준다.  

그리고 재배포를 하면  

![](/images/img_1116621bf7f2a770.png)

잘 될 것이다.  

![](/images/img_494bd172b91ddc04.png)

## 릴리즈 롤백하기

rollback 명령을 사용하면 이전 배포 버전으로 돌리는 것도 가능하다.  
그리 어렵지 않다.  

```javascript
helm rollback 릴리즈명
```

![](/images/img_ef440157f0372b01.png)
