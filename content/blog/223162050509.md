sqlx는 Rust의 대표적인 DB용 라이브러리 중 하나다.  
ORM은 아니고, 그냥 그.. 마이바티스 같은 쿼리 매퍼? 실행기다.  

ORM 같은 거창한 기능은 없다.  
그냥 쿼리에 구멍 뚫고 주입해서 쓰고, 반환값 좀 편하게 바인딩하고, async 기능이 잘 지원되는 것 정도다.  
간단하게 쓰기는 좋다.  

<br>

<br>

<br>

## 종속성

종속성은 다음과 같은 형태로 구성할 수 있다.  
tokio를 쓰고, postgresql를 쓸거라면 이렇게 구성하면 된다.  

```javascript
[dependencies]
tokio = { version = "1.21.0", features = ["full", "time"] }
sqlx = { version = "0.7", features = [ "runtime-tokio", "postgres" ] }
```

그리고 bind로 구멍에 값을 넣은 다음 fetch를 사용해서 실행, 값을 가져오면 된다.

## 컴파일타임 검증

컴파일타임 검증은 sqlx의 가장 강력한 기능 중 하나다.  
실제 DB 정보를 가지고서 쿼리를 검증하고, 타입과 일치하는지도 검증을 한다.  

예를 들어 이런 테이블이 있다고 하면  

![](/images/img_7115c1c011939d71.png)

테이블 리턴에 대한 타입을 정의하고  

![](/images/img_889b360e82a638e8.png)

이런식으로 사용할 수 있다.  

![](/images/img_00ece7e4a96903b9.png)

![](/images/img_eab90c8c613271d3.png)
그러면 저 타입을 기반으로 해서 자동으로 바인딩을 해줄 것이다.  

이 기능의 강력한 점은 쿼리와 리턴타입의 일치도도 검증을 해주고  

![](/images/img_900e1a061e2a785d.png)

심지어 실제 DB에 정의된 정보를 기반으로 컴파일타임에 검증을 해준다는 것이다.  
그래서 존재하지 않는 컬럼을 쿼리에 사용하면 에러를 뱉는다.  

![](/images/img_23b8efbcd840ad9c.png)
근데 이걸 어떻게 컴파일타임에 처리하는거지? 신기하기도 하다.
