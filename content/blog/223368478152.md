Cloudfront KeyValueStore는 Cloudfront Function을 보조하는 부가기능이다.  
Function에서 뭔가 데이터를 영속적으로 저장하고 꺼내쓸 필요가 있을때 사용한다.  

특수화된거라 public access 가능한 범용 DB는 아니고, CloudFront Function 엣지에서만 접근 가능하다.  

다른 데이터 스토리지들과 차별되는 장점은 대부분 리전의 모든 엣지 위치에 데이터가 분포되어서 최소한의 레이턴시로 빠르게 동작한다는 것이다.  

대표적인 사용례는 A/B 테스트 라우팅을 할때 분포 비율 등을 조정한다거나 하는 것이다.  
딱히 정형화된 사용방법이 있지는 않으니, 편할대로 쓰면 된다.

## A/B 라우팅 구성해보기

KeyValue 스토어를 기반으로 A/B 라우팅 확률 가중치를 조정하는 구성을 짜보도록 하겠다.  

![](/images/img_0bca0865038ccd32.png)

나는 /abtest라는 경로로 접속을 시도할때, 가중치에 따라 /abtest/a.html, /abtest/b.html 리소스에 라우팅을 시키려고 한다. (URI 리디렉션 없이)  

먼저 s3에 파일을 적절히 업로드했다.  

![](/images/img_9824e61baee97bdb.png)
별건없다.  

그리고 Cloudfront 함수를 만든 후에  

![](/images/img_4e8daf7628ddb4d7.png)

![](/images/img_0897fbab0afd5e9e.png)

![](/images/img_accb0b3b61a8dba4.png)
KeyValueStore를 연관시켜준다.  
그럼 저 함수에서 사용할 수 있는 고유 ID가 생성된다.  

함수 사용법에 대해서는 별도 포스트를 참고한다.  
[https://blog.naver.com/sssang97/223368455944](https://blog.naver.com/sssang97/223368455944)  

<br>

![](/images/img_e65d2d59f8c65267.png)
우선 Store에 초기값을 설정해주고  

Function에 코드를 작성한다.  

```javascript
import cf from "cloudfront";

const kvsId = "...b00ac861e1d1";

const kvsHandle = cf.kvs(kvsId);

async function handler(event) {
  const request = event.request;

  if (request.uri == "/abtest") {
    try {
      let aWeight = await kvsHandle.get("a_weight");
      let bWeight = await kvsHandle.get("b_weight");

      aWeight = aWeight ? parseInt(aWeight) : 0;
      bWeight = bWeight ? parseInt(bWeight) : 0;

      const percentOfA = aWeight / (aWeight + bWeight);

      const newUri = Math.random() < percentOfA ? "/abtest/a.html" : "/abtest/b.html";

      request.uri = newUri;
      return request;
    } catch (err) {
      console.log(`${request.uri} | ${err}`);
      return request;
    }
  } else {
    return request;
  }
}
```
가중치에 따라 확률로 매칭시켜주는 간단한 코드다.  
/abtest 경로로 들어올때만 특별 처리를 해줬다.  

함수 코드를 배포하고,
