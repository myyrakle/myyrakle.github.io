## 디버그 설정 구성

먼저 실행 구성을 위한 launch.json 파일을 생성한다.  

![](/images/img_069f142c5e439542.png)

<br>

![](/images/img_648a9d3c56bd600c.png)
만약 이런 구조를 갖고 있고, 디버그에서 저기에 있는 main을 실행시키고 싶다면  

이런 느낌으로 구성하면 된다.  

![](/images/img_b253e04199d75520.png)

```javascript
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Go Debug",
            "type": "go",
            "request": "launch",
            "mode": "debug",
            "program": "cmd/aims",
            "cwd": "${workspaceFolder}"
        }
    ]
}
```
cwd는 실행 컨텍스트인데, 저건 현재 프로젝트의 루트 패스를 실행 컨텍스트로 잡겠단 것이다. 이게 없으면 루트 패스 기준으로 참조하던 파일들을 참조하지 못하게 된다.  
program은 main이 존재하는 패키지 디렉터리다.  

<br>

![](/images/img_cb50a1d478fda405.png)
잘 설정하면 이렇게 구성이 좌측에 뜰 것이고  

실행하면  

![](/images/img_3e8d6f47e096b47d.png)

![](/images/img_aa7e927d5d2952d2.png)
디버그 모드로 잘 실행이 될 것이다.
