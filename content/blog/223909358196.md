OS 깔았는데 나중에 파티션 크기를 잘못 잡은걸 알고 다시 늘리고 싶을 수도 있다.  
내가 그랬다. 무지성으로 깔고 보니 4테라짜리 디스크에서 100기가만 파티션으로 잡아놨다.  

![](/images/img_388de970d59fdd3b.png)

일단 실수하지 않게 볼륨 정보들을 확인해보자  

```javascript
sudo pvs # PV(Physical Volume) 확인
sudo vgs # VG(Volume Group) 확인
sudo lvs # LV(Logical Volume) 확인
```

![](/images/img_708342ff42886e2a.png)

![](/images/img_7334a41f71135fd4.png)

![](/images/img_8aa705ab6884b6c4.png)
내가 잡고 싶은건 저 4테라짜리 vg-1이란 볼륨이다. 현재는 저기서 100기가 정도만 파티션으로 잡아버려서 나머지 3900기가 정도를 그냥 낭비하고 있는 상태다.  

이럴때는 LVM을 통해서 파티션을 재조정하는 것이 가능하다. 재부팅도 필요없다.  
파티션을 확장하고  

```javascript
# parted로 파티션 테이블 확인
sudo parted /dev/sda print

# LVM 파티션 확장 (보통 파티션 3번이 LVM)
sudo parted /dev/sda resizepart 3 100%
```

![](/images/img_928eb6db921d462f.png)

![](/images/img_fca71b1e3797222b.png)

피지컬 볼륨을 조정하고  

```javascript
# PV 크기 조정
sudo pvresize /dev/sda3
```

![](/images/img_f25b80d58492f6f2.png)

로지컬 볼륨까지 확장한다.  

```javascript
# LV 확장 (모든 여유 공간 사용)
sudo lvextend -l +100%FREE /dev/ubuntu-vg-1/ubuntu-lv
```

![](/images/img_c2d83ca05d127b07.png)

그리고 리사이즈를 돌리면  

```javascript
# ext4 파일시스템 크기 조정
sudo resize2fs /dev/ubuntu-vg-1/ubuntu-lv
```

![](/images/img_0b367b02f37ab001.png)

꽉 채워 맞춰서 늘어날 것이다.  

![](/images/img_65bde84a483c977e.png)
그럼 됐다.
