외부 네트워크 접속 없이 특정 리소스를 바라보며 처리를 하는 consumer를 만들게 됐다.  
이 케이스는 인스턴스가 무조건 하나만 있어야 하고, 다운타임이 조금은 있어도 상관이 없다.  

그래서 서버처럼 블루-그린 배포도 없이 레거시하게 stop and start 배포를 구성해봤다.  

언어는 nodejs를 쓰고, pm2로 단일 프로세스를 관리하도록 할 것이다.

## EC2 세팅

기본 런타임을 깔아주고  

```javascript
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | bash
. ~/.nvm/nvm.sh
nvm install 16
npm i -g pm2
```

pm2가 잘 도는지 본다.  

```javascript
pm2 start npm --name "consumer" -- start
```

## Code Deploy agent 설치

EC2에서 code deploy의 동작을 인지하고 상호작용을 하려면, 그런걸 처리하는 데몬이 깔려있어야 한다.  
그게 code deploy agent다. 깔아준다.  

```javascript
sudo yum update
sudo yum install ruby
sudo yum install wget
cd /home/ec2-user
wget https://bucket-name.s3.region-identifier.amazonaws.com/latest/install
chmod +x ./install
sudo ./install auto
sudo service codedeploy-agent status
```

![](/images/img_24596f64bb7699d6.png)
그래서 이렇게 실행되면 된다.

## Code build 생성

배포 환경들은 대충 기본적인 구성으로만 줬고  

![](/images/img_a47aaf92b07b1bea.png)

buildspec만 잘 달아줬다.  

![](/images/img_a6c49f0504f33691.png)

buildspec에서는 환경변수만 다운받고 바로 export하도록 구성했다.  

```javascript
version: 0.2

phases:
  pre_build:
    commands:
      - echo foo
  build:
    commands:
      - echo Build started on `date`
      - echo Building the Docker image...
      - aws s3 cp s3://.../prod/.env .env
  post_build:
    commands:
      - echo Build completed on `date`
      - echo success

artifacts:
  files:
    - '**/*'
    - '*'
```

## appspec 세팅

buildspec이 빌드에 대한 설정파일이라면, appspec은 배포에 대한 설정파일이다.  
이건 무조건 루트경로에 있어야 한다.  

이런식으로 작성할 수 있다.  

```javascript
version: 0.0
os: linux
files:
  - source: /
    destination: /home/ec2-user/app
    overwrite: true
hooks:
  ApplicationStop:
    - location: scripts/stop.sh
      runas: ec2-user
    - location: scripts/clear.sh
      runas: root
  Install:
    - location: scripts/install.sh
      runas: root
  ApplicationStart:
    - location: scripts/restart.sh
      runas: ec2-user
```
files는 codebuild에서 받아온 파일들을 어떻게 EC2에 집어넣을지에 대한 부분이다. 난 /home/ec2-user/app라는 경로에 복붙하도록 했다.  

그리고, hooks는 각 이벤트에 맞춰서 어떤 동작을 실행할지 지정하는 부분이다. 쉘코드를 프로젝트에 작성해둔 다음에, 그 경로를 넣어주면 된다.  
개인적으로 난 이게 좀 많이 귀찮고 짜증났다.  

세팅하기전에 싹 날리고, install해서 pm2 restart를 때리도록 했다.  

![](/images/img_4a8481745a62efdc.png)
이게 node 관련해서 환경변수가 꼬일때가 많아서 절대경로를 다 때려박았다.
