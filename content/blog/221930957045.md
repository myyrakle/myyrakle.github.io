서블릿이나 jsp를 다루어봤다면 war이라는 파일을 자주 보았을 것이다.  
그리고 자바를 다루어본 사람이라면 jar 파일은 더 많이 보았을 것이다.  

대관절 이 둘의 차이가 무엇일까?  

<br>

<br>

## jar?
jar은 Java ARchive의 축약으로, 하나의 자바 프로젝트를 통째로 압축해 만든 자바 전용 포맷이다.  
그래서 실제 구조는 zip과 큰 차이는 없다.  
jar은 하나의 완성된 라이브러리일 수도 있고, java.exe로 바로 실행 가능한 프로그램일 수도 있다.  

<br>

<br>

## war?
war은 Web Application Archive의 축약이다. 웹어플리케이션에 특화된 jar의 한 종류라고 보면 된다.  
주로 서블릿이나 jsp를 사용할 때 이 포맷으로 결과물을 배출한다.  
보통 소스뿐만 아니라 html, css 등의 정적 리소스들도 포함한다. 게다가 WEB-INF와 METH-INF라는 정의된 구조가 강요된다.  

그리고 jar을 실행할 때는 자바 런타임만 있으면 되지만, war을 실행하기 위해서는 tomcat 등의 서버 런타임도 필요하다.
